# research: domain terms

## objective

coin terms for domain objects that:
- maximize specificity — eliminate ambiguity
- maximize intuition — eliminate friction
- create a ubiquitous language

---

## entity terms

### DeclaredSquarespaceDomainRegistration

**chosen term**: `DeclaredSquarespaceDomainRegistration`

**decomposition**:
- `Declared` — follows declastruct pattern (declared desired state)
- `Squarespace` — provider namespace
- `Domain` — core concept (internet domain name)
- `Registration` — the act of register a domain with a registrar

**alternatives considered**:

| term | why rejected |
|------|--------------|
| SquarespaceDomain | too generic, no "Declared" prefix |
| DomainRecord | conflicts with DNS record concept |
| DomainAccount | implies user account, not domain |
| DomainLicense | not standard registrar terminology |

**parallel concepts**:
- **real estate**: property registration, deed
- **vehicles**: vehicle registration, title
- **healthcare**: patient registration
- **legal**: trademark registration

**citation [1]**: ICANN glossary
> "Registrant: The person or entity that registers a domain name."
> source: https://www.icann.org/resources/pages/glossary-2017-10-10-en

### DeclaredSquarespaceDomainDnsRecord

**chosen term**: `DeclaredSquarespaceDomainDnsRecord`

**decomposition**:
- `Declared` — declastruct pattern
- `Squarespace` — provider namespace
- `Domain` — parent context (which domain)
- `DnsRecord` — industry standard term for DNS entries

**alternatives considered**:

| term | why rejected |
|------|--------------|
| DnsEntry | "record" is more standard |
| ZoneRecord | implies zone file, too technical |
| NameRecord | ambiguous |
| DomainHost | conflicts with host attribute |

**parallel concepts**:
- **postal**: address record, relay instructions
- **phone directory**: entry
- **navigation**: waypoint, coordinate

**citation [2]**: RFC 1035
> "A resource record (RR) is the basic unit of DNS data."
> source: https://tools.ietf.org/html/rfc1035

### DeclaredSquarespaceDomainTransferRequest

**chosen term**: `DeclaredSquarespaceDomainTransferRequest`

**decomposition**:
- `Declared` — declastruct pattern
- `Squarespace` — provider namespace
- `Domain` — what is transferred
- `TransferRequest` — the formal request to transfer

**alternatives considered**:

| term | why rejected |
|------|--------------|
| DomainTransfer | ambiguous (could be in or out) |
| TransferCode | the code is a detail, not the whole |
| Exodus | too dramatic |
| Migration | implies data move, not registrar change |

**parallel concepts**:
- **real estate**: title transfer, deed transfer
- **finance**: wire transfer request
- **employment**: transfer request (between departments)
- **vehicles**: title transfer

**citation [3]**: ICANN transfer policy
> "Transfer: The process of move a domain name registration between registrars."
> source: https://www.icann.org/resources/pages/transfer-policy-2016-06-01-en

## literal terms

### DeclaredSquarespaceDomainRegistrationStatus

**values**: ACTIVE, EXPIRED, PENDING

**rationale**: standard registrar status terminology

**citation [4]**: ICANN registrar documentation
> "Status codes indicate the current state of a domain registration."
> source: https://www.icann.org/resources/pages/epp-status-codes-2014-06-16-en

### DeclaredSquarespaceRegistrarType

**values**: SQUARESPACE_DOMAINS_LLC, SQUARESPACE_DOMAINS_II_LLC, TUCOWS, KEY_SYSTEMS

**rationale**: exact legal entity names as registrars

**citation [5]**: ICANN accredited registrar list
> "Squarespace Domains LLC is an ICANN-accredited registrar."
> source: https://www.icann.org/registrar-reports/accredited-list.html

### DeclaredSquarespaceDomainDnsRecordType

**values**: A, AAAA, ALIAS, CNAME, MX, TXT, SRV, NS, CAA, DS, DNSKEY

**rationale**: RFC-defined DNS record type mnemonics

**citation [6]**: RFC 1035 / various RFCs
> "A records map hostnames to IPv4 addresses. AAAA records map to IPv6. CNAME creates aliases."
> source: https://tools.ietf.org/html/rfc1035

### DeclaredSquarespaceDomainLockReason

**values**: REGISTRATION_LOCK_60_DAY, TRANSFER_LOCK_60_DAY, CONTACT_UPDATE_LOCK

**rationale**: describes ICANN-mandated lock scenarios

**citation [7]**: ICANN transfer policy
> "Registrars may deny a transfer if the domain is within 60 days of initial registration or a recent transfer."
> source: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

### DeclaredSquarespaceTransferRequestStatus

**values**: REQUESTED, CODE_SENT, IN_PROGRESS, COMPLETED, CANCELLED

**rationale**: follows transfer lifecycle phases

## name conventions

### prefix pattern: Declared{Provider}{Domain}{Concept}

| segment | purpose |
|---------|---------|
| Declared | declastruct namespace (declarative control) |
| Squarespace | provider namespace |
| Domain | sub-namespace for domain-related entities |
| {Concept} | the specific entity or literal |

### suffix patterns

| suffix | definition |
|--------|------------|
| Registration | root entity (the registered domain) |
| Record | data entry (DNS, etc.) |
| Request | action entity (transfer) |
| Status | state literal |
| Type | classification literal |
| Reason | explanation literal |

## parallel domain wisdom

### real estate (oldest)

- **registration** → deed record
- **transfer** → title transfer
- **lock** → title hold / lien
- **DNS records** → property survey / plot records

### postal (centuries old)

- **domain** → address
- **DNS records** → relay instructions
- **transfer** → address change

### finance

- **transfer request** → wire transfer request
- **authorization code** → route number + account verification
- **lock** → account freeze

### vehicles

- **registration** → vehicle registration
- **transfer** → title transfer
- **status** → registration status (active, expired, suspended)

---

## summary

the chosen terms follow industry standard terminology:
- "registration" for domain ownership
- "record" for DNS entries
- "transfer request" for move operations

the `Declared` prefix aligns with declastruct patterns.
the `Squarespace` segment provides clear provider namespace.

no novel coinages — all terms are industry-standard domain registrar vocabulary.
