# research: domain terms for declastruct-squarespace

## objective

propose naming options for domain objects required to fulfill the vision. analyze current naming against industry terminology and parallel domains.

---

## current domain object naming

### entities

| current name | purpose |
|--------------|---------|
| DeclaredSquarespaceDomainRegistration | domain registered with squarespace |
| DeclaredSquarespaceDomainDnsRecord | dns record for a domain |
| DeclaredSquarespaceDomainTransferRequest | request to transfer domain out |

### literals

| current name | purpose |
|--------------|---------|
| DeclaredSquarespaceRegistrarType | backend registrar (TUCOWS, KEY_SYSTEMS, etc.) |

| DeclaredSquarespaceDomainDnsRecordType | dns record type (A, CNAME, MX, etc.) |
| DeclaredSquarespaceTransferRequestStatus | transfer request state |
| DeclaredSquarespaceDomainRegistrationStatus | domain registration state |

## industry terminology analysis

### domain transfer terminology

| term | context | definition |
|------|---------|------------|
| EPP code | ICANN standard | extensible provisioning protocol authorization code |
| auth code | common usage | authorization code for inter-registrar transfer |
| transfer code | user-faced | same as auth code, simpler language |
| authorization code | formal | ICANN's official term |

**recommendation:** use "authorization code" or "auth code" — both are ICANN-recognized [1]

### domain lock terminology

| term | context | definition |
|------|---------|------------|
| registrar lock | common usage | prevents unauthorized transfers |
| transfer lock | specific | lock that prevents transfers |
| client transfer prohibited | EPP status | formal status code |
| domain lock | user-faced | generic term for any lock |

**recommendation:** "isLocked" is clear; "lockReason" covers specifics [2]

### dns terminology

| term | standard | definition |
|------|----------|------------|
| resource record | RFC 1035 | formal term for dns entry |
| dns record | common usage | user-friendly term |
| zone record | dns context | record within a zone file |

**recommendation:** "DnsRecord" is standard and clear [3]

### dnssec terminology

| term | definition |
|------|------------|
| DS record | delegation signer record at parent zone |
| DNSKEY | public key in zone |
| RRSIG | record signature |
| DNSSEC | domain name system security extensions |

**recommendation:** "dnssecEnabled" is clear; no need for deeper terms [4]

## parallel domain analysis

### real estate analogy

| real estate term | domain equivalent | insight |
|------------------|-------------------|---------|
| deed | domain registration | proof of ownership |
| title transfer | domain transfer | change of ownership |
| conveyance | transfer process | act of transfer |
| title search | whois lookup | verify ownership |
| escrow | registrar | intermediary that holds asset |

**insight:** "transfer" is correct term; "conveyance" too formal for this context [5]

### logistics analogy

| logistics term | domain equivalent | insight |
|----------------|-------------------|---------|
| chain of custody | registrar history | who held domain when |
| handoff | transfer | pass control |
| manifest | dns records | what's included |
| bill of lading | auth code | authorization to move |

**insight:** "handoff" could work but "transfer" is industry standard [6]

## name options analysis

### entity: domain registration

| option | pros | cons |
|--------|------|------|
| DeclaredSquarespaceDomainRegistration | matches domain concept | long |
| DeclaredSquarespaceDomain | shorter | loses "registration" precision |
| DeclaredSquarespaceHostedDomain | clarifies host | adds unnecessary word |

**recommendation:** keep `DeclaredSquarespaceDomainRegistration` — precision matters for transfer context

### entity: transfer request

| option | pros | cons |
|--------|------|------|
| DeclaredSquarespaceDomainTransferRequest | explicit | long |
| DeclaredSquarespaceTransferCode | matches outcome | code is emailed, not returned |
| DeclaredSquarespaceDomainConveyance | formal | unfamiliar term |
| DeclaredSquarespaceTransferOut | directional | loses "request" semantics |

**recommendation:** keep `DeclaredSquarespaceDomainTransferRequest` — "request" is accurate since code is emailed

### entity: dns record

| option | pros | cons |
|--------|------|------|
| DeclaredSquarespaceDomainDnsRecord | explicit | long |
| DeclaredSquarespaceDnsRecord | shorter | loses domain context |
| DeclaredSquarespaceZoneRecord | technical | unfamiliar to users |

**recommendation:** keep `DeclaredSquarespaceDomainDnsRecord` — domain context important for refs

### literal: registrar type

| option | pros | cons |
|--------|------|------|
| DeclaredSquarespaceRegistrarType | matches domain | none |
| DeclaredSquarespaceBackendRegistrar | explicit | adds "backend" |
| DeclaredSquarespaceRegistrarOfRecord | ICANN term | longer |

**recommendation:** keep `DeclaredSquarespaceRegistrarType` — clear and concise

### literal: lock reason

| option | pros | cons |
|--------|------|------|
| DeclaredSquarespaceDomainLockReason | explicit | none |
| DeclaredSquarespaceTransferRestriction | specific | loses other lock types |
| DeclaredSquarespaceLockType | generic | less descriptive |

**recommendation:** keep `DeclaredSquarespaceDomainLockReason` — "reason" clarifies why locked

## prefix analysis: "Declared" vs alternatives

| prefix | sense | pros | cons |
|--------|-------|------|------|
| Declared | declarative state | aligns with declastruct pattern | standard for this repo |
| Squarespace | platform origin | clear source | loses declarative intent |
| Managed | operational | implies control | overused |
| Controlled | operational | implies control | vague |

**recommendation:** keep `Declared` prefix — aligns with declastruct pattern used in vision

## term standardization

### terms to use consistently

| concept | standard term | avoid |
|---------|---------------|-------|
| authorization code | authorizationCode or authCode | epp code, transfer code |
| domain lock | isLocked + lockReason | clientTransferProhibited |
| dns record | DnsRecord | zone record, resource record |
| transfer | transfer | conveyance, handoff |
| registrar | registrar | provider, host |

### abbreviations

| full term | abbreviation | usage |
|-----------|--------------|-------|
| domain name system | dns | always lowercase in code |
| dns security extensions | dnssec | always lowercase in code |
| time to live | ttl | always lowercase in code |
| extensible provision protocol | epp | avoid — use "auth code" |

## summary

| category | recommendation |
|----------|----------------|
| entity names | keep current names — explicit, precise, domain-aligned |
| literal names | keep current names — clear, standard terminology |
| prefix | keep "Declared" — aligns with declastruct pattern |
| terminology | use "auth code" over "epp code"; "transfer" over "conveyance" |

**conclusion:** current names are well-aligned with industry standards and declastruct patterns. no changes recommended. terms are clear, avoid jargon, and match ICANN terminology where appropriate.
