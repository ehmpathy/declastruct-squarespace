# research: templates for declastruct-squarespace

## objective

identify template implementations that demonstrate declastruct patterns. extract key patterns and map to the wish.

---

## template: declastruct

### source

**repo:** https://github.com/ehmpathy/declastruct
**description:** "Add declarative control to any resource constructs. Declare, plan, and apply within an observable pit-of-success."

[1] gh repo view ehmpathy/declastruct

### key patterns

#### pattern: plan/apply workflow
```sh
npx declastruct plan --wish resources.ts --into plan.json
npx declastruct apply --plan plan.json
```

#### pattern: provider contract
```ts
export const getProviders = async () => [provider1, provider2];
export const getResources = async () => [...resources];
```

#### pattern: dao interface
```ts
interface DeclastructDao {
  get: {
    one: { byUnique(input): Promise<Resource | null> };
    all(input): Promise<Resource[]>;
  };
  set: {
    findsert?(input): Promise<Resource>;
    upsert?(input): Promise<Resource>;
    delete?(input): Promise<void>;
  };
}
```

### relation to wish

provides the cli and core framework for declarative control. squarespace provider plugs into this framework.

#### pattern: get operations

```ts
export const getCustomer = async (
  input: {
    by: PickOne<{
      primary: RefByPrimary<typeof DeclaredStripeCustomer>;
      unique: RefByUnique<typeof DeclaredStripeCustomer>;
      ref: Ref<typeof DeclaredStripeCustomer>;
    }>;
  },
  context: StripeApiContext,
): Promise<HasMetadata<DeclaredStripeCustomer> | null> => {
  if (input.by.ref)
    return isUniqueKeyRef({ of: DeclaredStripeCustomer })(input.by.ref)
      ? getCustomer({ by: { unique: input.by.ref } }, context)
      : getCustomer({ by: { primary: input.by.ref } }, context);
  // ...
};
```
[3] rule.prefer.declastruct.[demo].md from briefs

#### pattern: set operations with findsert/upsert

```ts
export const setCustomer = async (
  input: PickOne<{
    findsert: DeclaredStripeCustomer;
    upsert: DeclaredStripeCustomer;
  }>,
  context: StripeApiContext,
): Promise<HasMetadata<DeclaredStripeCustomer>> => {
  const customerFound = await getCustomer({ by: { unique: { email } } }, context);

  if (customerFound) {
    if (input.findsert) return customerFound;
    if (input.upsert) return castTo...(await context.stripe.customers.update(...));
  }

  return castTo...(await context.stripe.customers.create(...));
};
```
[3] rule.prefer.declastruct.[demo].md from briefs

### relation to wish

demonstrates declastruct pattern for api-based resources. squarespace differs by:
- uses playwright instead of api calls
- requires 2fa automation
- uses cache layer for remote state

## pattern summary

| pattern | stripe template | squarespace template | relation to wish |
|---------|-----------------|----------------------|------------------|
| domain entity with primary/unique | yes | yes | define domain registrations, dns records, transfer requests |
| cast from raw to entity | yes | yes | transform scraped data to typed entities |
| get.byUnique / get.byPrimary | yes | yes | lookup domains by name |
| get.all | yes | yes | enumerate all domains |
| set.findsert | yes | yes | idempotent transfer request |
| set.upsert | yes | yes | update lock/dnssec state |
| provider factory | yes | yes | create squarespace provider |
| context injection | yes | yes | pass auth, cache, browser config |
| rate limit | no | yes (bottleneck) | prevent bot detection |
| browser automation | no | yes (playwright) | automate ui-only platform |
| 2fa automation | no | yes (otpauth) | handle totp |
| remote state cache | partial | yes | cache scraped data |

---

## conclusion

the declastruct-stripe-sdk demonstrates the core declastruct patterns (entities, get/set operations, provider factory). the current repo extends these patterns with:
- playwright browser automation (replaces api calls)
- totp 2fa integration (no equivalent in stripe)
- bottleneck rate limit (more critical for ui automation)
- remote state cache (critical for scrape-heavy workflows)

all patterns from stripe template apply, with browser automation layer inserted between dao operations and remote state.
