# research: oss levers for declastruct-squarespace

## objective

identify open source tools that enable the domain transfer wish. evaluate maintenance status, adoption, and fit for purpose.

---

## tool: playwright

### what
browser automation library from microsoft. controls chromium, firefox, webkit via unified api.

### why relevant
squarespace has no api. playwright automates their web ui for domain operations.

### maintenance status
- **publisher:** microsoft
- **github stars:** 80k+
- **npm weekly downloads:** 32M+
- **last release:** active (multiple releases per month)
- **typescript support:** native

### adoption
used by: microsoft, adobe, shopify, slack, stripe, zoom

### how to use
```ts
import { chromium } from 'playwright';

const browser = await chromium.launch({ headless: true });
const page = await browser.newPage();
await page.goto('https://account.squarespace.com/domains');
await page.fill('[name="email"]', email);
await page.click('[type="submit"]');
```
[1] https://playwright.dev/docs/intro

### pros
- native typescript types
- auto-wait for elements
- cross-browser support
- stealth plugin available (playwright-stealth)
- parallel context support
- trace viewer for debug

### cons
- heavy dependency (~250MB chromium)
- ui changes break selectors
- requires headless browser runtime

### verdict: [USE]
primary automation layer. no viable alternative for ui-only platforms.

## tool: playwright-stealth

### what
plugin that patches playwright to reduce bot detection signals.

### why relevant
squarespace may detect automated browsers. stealth mode reduces fingerprint leakage.

### maintenance status
- **npm package:** `playwright-stealth` (unofficial fork)
- **github:** rebrowser/rebrowser-patches (active fork)
- **original:** berstend/puppeteer-extra-plugin-stealth (mature, maintained)
- **npm weekly downloads:** ~15k

### adoption
derived from puppeteer-extra stealth plugin, which is widely used for web scrape operations.

### how to use
```ts
import { chromium } from 'playwright';
import { addStealth } from 'playwright-stealth';

const browser = await chromium.launch();
const context = await browser.newContext();
await addStealth(context);
```
[2] https://github.com/berstend/puppeteer-extra/tree/master/packages/puppeteer-extra-plugin-stealth

### pros
- masks webdriver property
- patches navigator fingerprint
- reduces automated browser signals
- drop-in integration

### cons
- cat-and-mouse with detection systems
- not guaranteed against advanced detection
- third-party maintained

### verdict: [USE]
reduces bot detection risk. already in codebase.

## tool: otpauth

### what
one-time password library that implements HOTP (RFC 4226) and TOTP (RFC 6238).

### why relevant
squarespace 2fa requires totp codes. otpauth generates valid codes from secret.

### maintenance status
- **version:** 9.5.0
- **npm weekly downloads:** ~450k
- **last release:** 2024 (active)
- **typescript support:** native

### adoption
standard totp library for node.js applications. used for 2fa automation.

### how to use
```ts
import * as OTPAuth from 'otpauth';

const totp = new OTPAuth.TOTP({
  issuer: 'Squarespace',
  algorithm: 'SHA1',
  digits: 6,
  period: 30,
  secret: OTPAuth.Secret.fromBase32(totpSecret),
});
const code = totp.generate();
```
[3] https://www.npmjs.com/package/otpauth

### pros
- rfc compliant
- native typescript
- lightweight
- zero dependencies
- supports multiple algorithms

### cons
- requires user to extract totp secret from authenticator setup
- sms/passkey 2fa not supported (limitation of totp, not library)

### verdict: [USE]
required for 2fa automation. already in codebase.

## tool: bottleneck

### what
rate limiter and job scheduler. controls request concurrency and pace.

### why relevant
squarespace may rate-limit or detect rapid automated requests. bottleneck prevents this.

### maintenance status
- **version:** 2.19.5
- **npm weekly downloads:** ~3.5M
- **last publish:** 7 years ago (stable, feature-complete)
- **github issues:** minimal (mature)

### adoption
used by 998+ npm packages. standard rate limit solution.

### how to use
```ts
import Bottleneck from 'bottleneck';

const limiter = new Bottleneck({
  maxConcurrent: 1,
  minTime: 500,  // 500ms between operations
});

await limiter.schedule(() => scrapeDomainsList({ page }));
```
[4] https://www.npmjs.com/package/bottleneck

### pros
- simple api
- supports cluster mode
- priority queues
- retry support
- battle-tested

### cons
- no recent updates (but stable)
- in-memory only (no distributed state)

### verdict: [USE]
prevents rate-based detection. already in codebase.

## tool: domain-objects

### what
ddd library for domain entities and literals with validation, identity, and serialization.

### why relevant
declastruct pattern requires typed domain objects with primary/unique keys.

### maintenance status
- **publisher:** ehmpathy
- **version:** 0.31.9
- **npm weekly downloads:** ~2k
- **last release:** 2024 (active)
- **typescript support:** native

### adoption
used across ehmpathy organization repositories. standard for declastruct implementations.

### how to use
```ts
import { DomainEntity, RefByUnique } from 'domain-objects';

interface DeclaredSquarespaceDomainRegistration {
  name: string;
  isLocked: boolean;
}

class DeclaredSquarespaceDomainRegistration
  extends DomainEntity<DeclaredSquarespaceDomainRegistration>
{
  public static primary = ['name'] as const;
  public static unique = ['name'] as const;
}
```
[5] https://www.npmjs.com/package/domain-objects

### pros
- typed primary/unique keys
- RefByUnique/RefByPrimary for references
- serialization built-in
- validation hooks
- nested hydration

### cons
- ehmpathy-specific patterns
- curve to learn for new contributors

### verdict: [USE]
required for declastruct pattern. already in codebase.

## tool: with-simple-cache

### what
higher-order function for memoization with configurable cache backends.

### why relevant
browser instances and pages are expensive. cache enables reuse across operations.

### maintenance status
- **publisher:** ehmpathy
- **version:** 0.15.3
- **npm weekly downloads:** ~1k
- **last release:** 2024 (active)

### how to use
```ts
import { withSimpleCache, createInMemoryCache } from 'with-simple-cache';

const cache = createInMemoryCache<Browser>({ expiration: { hours: 1 } });
const cachedLaunch = withSimpleCache(launchBrowser, { cache });
```
[6] https://www.npmjs.com/package/with-simple-cache

### pros
- simple hof pattern
- multiple backends (in-memory, on-disk)
- configurable expiration
- typed

### cons
- ehmpathy-specific

### verdict: [USE]
enables browser/page reuse. already in codebase.

## tool: with-remote-state-cache

### what
cache layer for remote state with automatic invalidation on mutations.

### why relevant
squarespace scrape is slow. cache remote state to avoid redundant scrapes.

### maintenance status
- **publisher:** ehmpathy
- **version:** 1.3.12
- **npm weekly downloads:** ~500
- **last release:** 2024 (active)

### how to use
```ts
import { createRemoteStateCacheContext } from 'with-remote-state-cache';

const { withRemoteStateQueryCache, withRemoteStateMutationRegistration } =
  createRemoteStateCacheContext({
    cache: ({ fromInput }) => fromInput[1].agentOptions.remoteState.cache,
    serialize: { key: (input) => input[0].name },
  });

const getAllDomainsWithCache = withRemoteStateQueryCache(getAllDomains);
const setDomainWithInvalidation = withRemoteStateMutationRegistration(setDomain);
```
[7] https://www.npmjs.com/package/with-remote-state-cache

### pros
- automatic cache invalidation on mutations
- query/mutation separation
- trigger-based invalidation
- typed

### cons
- ehmpathy-specific
- curve to learn

### verdict: [USE]
critical for 300+ domain accounts. already in codebase.

## tool: simple-on-disk-cache

### what
file-based cache with ttl expiration. persists across process restarts.

### why relevant
batch operations benefit from disk-persisted cache.

### maintenance status
- **publisher:** ehmpathy
- **version:** 1.7.3
- **npm weekly downloads:** ~500
- **last release:** 2024 (active)

### how to use
```ts
import { createOnDiskCache } from 'simple-on-disk-cache';

const cache = createOnDiskCache({
  directory: { mounted: { path: '.cache/squarespace' } },
  expiration: { hours: 24 },
});
```
[8] https://www.npmjs.com/package/simple-on-disk-cache

### pros
- survives process restarts
- configurable expiration
- json serialization
- typed

### cons
- disk i/o overhead
- cleanup responsibility

### verdict: [USE]
enables cache persistence for batch workflows. already in codebase.

## alternative tools considered

### puppeteer (not used)

| aspect | playwright | puppeteer |
|--------|-----------|-----------|
| publisher | microsoft | google |
| cross-browser | yes | chromium-focused |
| api | modern, auto-wait | older patterns |
| stealth | plugin available | plugin available |

**decision:** playwright has better api and microsoft support. no reason to switch.

### selenium (not used)

| aspect | playwright | selenium |
|--------|-----------|----------|
| language | node.js native | java-centric, node bindings |
| speed | fast | slower |
| setup | simple | requires webdriver |

**decision:** selenium adds complexity without benefit.

### speakeasy (not used for totp)

| aspect | otpauth | speakeasy |
|--------|---------|-----------|
| maintenance | active | stale (last update 2019) |
| typescript | native | @types needed |
| api | modern | older |

**decision:** otpauth is actively maintained with native types.
