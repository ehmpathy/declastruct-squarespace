# research: remote access

## remote repositories identified

### 1. squarespace domain management web ui

**endpoint**: `https://account.squarespace.com/domains`

**type**: web application (no api available)

**authentication required**: yes — email, password, 2fa (totp/sms/passkey)

**citation [1]**: squarespace help documentation
> "Squarespace doesn't have a public API for managing domain registrations. All domain management must be done through the web interface."

## best practices

### 1. sequential operation with delays

**citation [4]**: squarespace rate limit behavior (observed)
> rapid sequential requests may trigger bot detection or rate limits. sequential operation with randomized delays (500-2000ms) between actions reduces detection risk.
> source: empirical observation from current implementation

### 2. session persistence

**pattern**: reuse browser context across operations

**citation [5]**: playwright documentation on contexts
> "Browser contexts provide a way to operate multiple independent browser sessions... Each context has isolated storage, cookies, and state."
> source: https://playwright.dev/docs/browser-contexts

### 3. remote state cache

**pattern**: cache scraped state to reduce repeated scrape operations

**libraries**:
- `with-remote-state-cache` — declarative cache wrapper
- `simple-on-disk-cache` — filesystem cache store

**citation [6]**: with-remote-state-cache readme
> "A wrapper that adds caching to any function that retrieves remote-state resources, with declarative cache invalidation, updates, and optimistic updates."
> source: https://github.com/ehmpathy/with-remote-state-cache

### 4. authentication flow

**sequence**:
1. navigate to login page
2. enter email + password
3. handle 2fa challenge (totp via `otplib`)
4. store session cookies for reuse

**citation [7]**: otplib documentation
> "otplib is a JavaScript One Time Password (OTP) library... It implements both HOTP and TOTP."
> source: https://github.com/yeojz/otplib

   - `generateTotpCode.ts` — totp code generation
   - `handleReauthentication.ts` — session refresh

3. **cache layer**: `src/infra/performance/`
   - `withRemoteStateCache.ts` — cache wrapper
