# research: oss levers (production codepaths)

## browser automation

### playwright

**package**: `playwright` v1.49.1
**source**: https://github.com/microsoft/playwright
**maintained by**: microsoft

**citation [1]**: playwright readme
> "Playwright enables reliable end-to-end test for modern web apps. Any browser. Any platform. One API."
> source: https://github.com/microsoft/playwright

**use case**: automate squarespace web ui (no api available)

**pros**:
- cross-browser support (chromium, firefox, webkit)
- auto-wait for elements
- network interception
- storage state persistence
- headless and headful modes
- active maintenance by microsoft

**cons**:
- chromium download size (~200MB)
- requires system dependencies on linux

**frontier adopters**: microsoft, google, netlify

### playwright-extra

**package**: `playwright-extra` v4.3.6
**source**: https://github.com/berstend/puppeteer-extra

**citation [2]**: playwright-extra readme
> "A modular plugin framework for playwright to enable cool plugins through a clean interface."
> source: https://github.com/berstend/puppeteer-extra/tree/master/packages/playwright-extra

**use case**: extend playwright with stealth plugins

**pros**:
- plugin architecture
- compatible with puppeteer-extra plugins
- same api as playwright

**cons**:
- additional dependency layer
- plugin compatibility varies

### puppeteer-extra-plugin-stealth

**package**: `puppeteer-extra-plugin-stealth` v2.11.2
**source**: https://github.com/berstend/puppeteer-extra

**citation [3]**: stealth plugin readme
> "Stealth plugin for puppeteer-extra and playwright-extra. Makes it harder for websites to detect that the browser is controlled by puppeteer/playwright. Apply various evasion techniques to hide automation signals."
> source: https://github.com/berstend/puppeteer-extra/tree/master/packages/puppeteer-extra-plugin-stealth

**use case**: evade bot detection on squarespace

**evasion techniques**:
- navigator.webdriver = false
- remove HeadlessChrome from user agent
- spoof webgl renderer
- patch plugins and fonts fingerprints

**pros**:
- comprehensive evasion patches
- actively maintained
- works with playwright-extra

**cons**:
- cat-and-mouse with bot detection updates
- no guarantee against all detection

## authentication

### otpauth

**package**: `otpauth` v9.3.5
**source**: https://github.com/hectorm/otpauth

**citation [4]**: otpauth readme
> "One Time Password (HOTP/TOTP) library for Node.js, Deno and browsers."
> source: https://github.com/hectorm/otpauth

**use case**: generate TOTP codes for squarespace 2fa

**example**:
```ts
import { TOTP } from 'otpauth';
const totp = new TOTP({ secret: 'BASE32SECRET' });
const code = totp.generate();
```

**pros**:
- rfc 4226/6238 compliant
- zero dependencies
- works in browser and node

**cons**:
- none significant

## rate limit

### bottleneck

**package**: `bottleneck` v2.19.5
**source**: https://github.com/SGrondin/bottleneck

**citation [5]**: bottleneck readme
> "Bottleneck is a lightweight and zero-dependency Task Scheduler and Rate Limiter for Node.js and the browser."
> source: https://github.com/SGrondin/bottleneck

**use case**: sequential operation with delays between squarespace ui interactions

**example**:
```ts
import Bottleneck from 'bottleneck';
const limiter = new Bottleneck({ maxConcurrent: 1, minTime: 500 });
await limiter.schedule(() => scrapeOneDomain(page));
```

**pros**:
- zero dependencies
- flexible rate limit strategies
- reservoir for daily/hourly limits
- cluster support (redis)

**cons**:
- none significant for this use case

## cache

### simple-in-memory-cache

**package**: `simple-in-memory-cache` v0.4.0
**source**: https://github.com/ehmpathy/simple-in-memory-cache

**citation [6]**: simple-in-memory-cache readme
> "A simple in-memory cache with TTL support."
> source: https://github.com/ehmpathy/simple-in-memory-cache

**use case**: cache browser session to avoid repeated login

**pros**:
- simple api
- ttl support
- typed

**cons**:
- memory-only (lost on restart)

### simple-on-disk-cache

**package**: `simple-on-disk-cache` v1.7.3
**source**: https://github.com/ehmpathy/simple-on-disk-cache

**citation [7]**: simple-on-disk-cache readme
> "A simple on-disk cache, perfect for cache remote-state locally for development or production use."
> source: https://github.com/ehmpathy/simple-on-disk-cache

**use case**: persist scraped domain data between runs

**pros**:
- survives restarts
- configurable directory
- ttl support
- typed

**cons**:
- disk i/o overhead

### with-remote-state-cache

**package**: `with-remote-state-cache` v1.3.12
**source**: https://github.com/ehmpathy/with-remote-state-cache

**citation [8]**: with-remote-state-cache readme
> "A wrapper that adds cache to any function that retrieves remote-state resources, with declarative cache invalidation, updates, and optimistic updates."
> source: https://github.com/ehmpathy/with-remote-state-cache

**use case**: wrap scraper operations with automatic cache

**pattern**:
```ts
const cachedGetDomains = withRemoteStateCache(
  getAllDomains,
  { ttl: 60000 }
);
```

**pros**:
- declarative cache invalidation
- optimistic updates
- composable

**cons**:
- requires cache store (in-memory or on-disk)

## domain model

### domain-objects

**package**: `domain-objects` v0.31.7
**source**: https://github.com/ehmpathy/domain-objects

**citation [9]**: domain-objects readme
> "Represent domain objects, leverage domain knowledge, and add runtime validation. Supports domain driven design patterns like DomainEntity, DomainLiteral, and reference types."
> source: https://www.npmjs.com/package/domain-objects

**use case**: model squarespace entities (domains, dns records, transfer requests)

**features**:
- DomainEntity for identity-based objects
- DomainLiteral for value objects
- RefByUnique / RefByPrimary for typed references
- unique / primary / readonly declarations

**pros**:
- ddd patterns out of box
- typed references
- clone/serialize utilities

**cons**:
- curve for ddd concepts
