# research: captcha handle and bot detection bypass

researched: 2026-03-10

## summary

this research covers strategies for cloudflare turnstile captcha handle and bot detection bypass in playwright automation, with focus on:
- tiered approach: stealth → **headful checkbox click** → human fallback
- session persistence enables headless/headful browser pivot
- **key finding**: managed checkbox can be clicked in headed mode (no OSS solver needed)
- OSS solvers exist but typically unnecessary for checkbox mode

---

## cloudflare turnstile overview

### widget types

cloudflare turnstile offers three widget modes [1]:

| type | user interaction | detection level |
|------|------------------|-----------------|
| **managed** | checkbox if suspected bot | medium |
| **non-interactive** | widget visible, no click needed | low |
| **invisible** | completely hidden | low |

managed mode selects a challenge based on visitor browser signals and presents an interactive checkbox only if it detects potentially automated traffic [1].

### detection signals

cloudflare uses multiple detection layers [2][3]:
- TLS fingerprint (cipher suites, HTTP/2 headers)
- browser fingerprint (canvas, WebGL, audio context)
- behavioral analysis (mouse movements, click patterns)
- IP reputation score
- CDP (chrome devtools protocol) command detection

the CDP detection is critical: popular automation libraries rely on `Runtime.Enable` which reveals browser is controlled by automation software [4].

## tier 1: stealth plugins

### playwright-extra + puppeteer-extra-plugin-stealth

the stealth plugin applies evasion patches to reduce bot detection [5][6]:
- sets `navigator.webdriver` to `false`
- removes `HeadlessChrome` from user agent
- spoofs WebGL renderer
- patches plugins and fonts fingerprints
- overrides navigator languages

**limitation**: only supports chromium, not firefox or webkit [5].

**effectiveness**: passes fpscanner, intoli, and areyouheadless tests, but detection is a cat-and-mouse game [6].

```ts
import { chromium } from 'playwright-extra';
import StealthPlugin from 'puppeteer-extra-plugin-stealth';

chromium.use(StealthPlugin());
const browser = await chromium.launch({ headless: true });
```

sources: [5][6][7][8]

### patchright (undetected playwright)

patchright provides patched, undetected drop-in replacements for playwright [9][10].

**key innovation**: patches playwright at source level to avoid use of `Runtime.Enable` entirely, runs JavaScript in isolated ExecutionContexts instead [10].

**passes tests**: brotector, cloudflare, kasada, akamai, shape/f5, bet365, datadome, fingerprint.com, creepjs, sannysoft [9].

**limitation**: only supports chromium [9].

```sh
npm install patchright
```

```ts
import { chromium } from 'patchright';
const browser = await chromium.launch({ headless: true });
```

sources: [9][10][11]

### rebrowser-patches

collection of patches for puppeteer and playwright to avoid automation detection [4][12].

**the problem**: `Runtime.Enable` CDP command triggers anti-bot detection.

**the solution**: disables automatic `Runtime.Enable`, manually creates contexts with unknown IDs [4].

**drop-in packages**: `rebrowser-playwright`, `rebrowser-playwright-core` [12].

sources: [4][12][13]

### camoufox (anti-detect browser)

open-source fingerprint evasion library built on firefox [14][15].

**key feature**: modifies device information in C++ implementation instead of JavaScript injection, which leaves no trail [15].

**playwright compatible**: uses same API with different initialization [15].

**limitation**: still in active development; bug fixes may be slow (developer medical emergency in early 2025) [15].

sources: [14][15][16][17]

## tier 1.5: headful checkbox click (stealth + managed widget)

### key finding

for cloudflare turnstile **managed** mode (the "verify you are human" checkbox), stealth plugins alone are insufficient in headless mode. the checkbox can be clicked successfully only with **headed browser + stealth** [38][39][40][41][42].

| configuration | checkbox click success | notes |
|---------------|------------------------|-------|
| vanilla playwright headless | NO | bot detected instantly |
| stealth plugin headless | NO | still detected despite patches |
| patchright headless | NO | CDP patches not enough |
| **patchright + headed** | **YES** | works reliably [38][39] |
| **stealth + headed** | **YES** | works reliably |

### why headless fails

headless browsers expose detectable signals that current stealth patches cannot mask [40][41]:

1. **window focus state** — headless cannot simulate real window focus events
2. **rendering differences** — canvas/WebGL render differently in headless
3. **browser event patterns** — interaction events fire differently without display
4. **CDP artifacts** — even with patchright, some CDP traces persist in headless
5. **cloudflare behavior modeling** — their ML models trained on headless vs headed patterns

cloudflare's managed checkbox challenge specifically targets these signals [42]. the checkbox appears simple but performs deep browser environment analysis before validation.

### implication for architecture

this finding changes the tier 2 approach:

**before**: tier 2 = turnaround OSS solver (complex, limited lifespan)

**after**: tier 2 = headful browser click (simple, reliable)

the captcha pivot pattern becomes:
```
1. detect turnstile challenge
2. save storageState
3. close headless browser
4. launch headed browser with storageState
5. click checkbox (automated, not human)
6. save updated storageState
7. close headed browser
8. resume headless with fresh session
```

this eliminates the need for OSS solvers like turnaround for managed checkbox mode.

**exception**: if turnstile presents an interactive puzzle (rare for low-risk sessions), human fallback still needed.

sources: [38][39][40][41][42]

## tier 2: auto-solve (OSS solvers)

### turnaround

browser-based turnstile solver [18].

```python
from playwright.sync_api import sync_playwright
from turnaround import Solver

solver = Solver()
token = solver.solve(url="https://example.com", sitekey="0x...")
```

**limitation**: OSS nature means cloudflare can study the code; limited lifespan [19].

sources: [18][19]

### turnaround-api

cloudflare turnstile solve API based on turnaround, built with python + flask [20].

### theyka/turnstile-solver

python-based solver with patchright library and multi-threaded execution [21].

**solve time**: 4-6 seconds per captcha [21].

### cloudflare-solver

async python solution with humanization and proxy support [22].

sources: [20][21][22]

## tier 4: human fallback (captcha pivot)

### session persistence via storageState

playwright's `storageState` method captures and persists browser session data (cookies, localStorage, IndexedDB) [28][29].

```ts
// save session
const storageState = await context.storageState();
fs.writeFileSync('session.json', JSON.stringify(storageState));

// restore session
const context = await browser.newContext({
  storageState: JSON.parse(fs.readFileSync('session.json'))
});
```

**performance**: 71% faster test execution with authentication state reuse [30].

sources: [28][29][30][31]

### persistent context approach

uses `userDataDir` to retain data between test runs [32]:

```ts
const context = await chromium.launchPersistentContext('./user-data', {
  headless: false,
});
```

sources: [32]

### headless/headful switch

playwright supports runtime mode switch [33][34]:

```ts
// headless (default)
const browser = await chromium.launch({ headless: true });

// headful (for debug or captcha solve)
const browser = await chromium.launch({ headless: false });
```

**strategy**: automate in headless, switch to headful for captcha solve [34].

sources: [33][34][35]

### captcha pivot pattern

when captcha detected:

```
1. detect captcha (turnstile iframe or challenge page)
2. save storageState to disk
3. close headless browser
4. launch headful browser with same storageState
5. navigate to same URL
6. human solves captcha
7. save updated storageState
8. close headful browser
9. resume headless with fresh session
```

this pattern leverages session persistence to transfer authentication state between browser instances.

## squarespace-specific considerations

### cloudflare integration

squarespace sites can optionally use cloudflare for additional protection [36]:
- bot fight mode (included with plan)
- super bot fight mode
- managed challenges

most squarespace sites don't need cloudflare as squarespace provides built-in SSL and security [36].

### detection risk factors

| risk | mitigation |
|------|------------|
| headless user agent | stealth plugin patches |
| CDP detection | patchright or rebrowser-patches |
| rapid sequential operations | bottleneck rate limit (500ms+) |
| datacenter IP | residential proxies if needed |
| absent browser plugins | stealth plugin spoofs |

## recommended tiered approach

### tier 1: prevention (stealth)

install and configure stealth plugin:

```sh
npm install playwright-extra puppeteer-extra-plugin-stealth
```

**expected outcome**: most automation passes without captcha trigger.

### tier 2: headful checkbox click

if turnstile managed checkbox detected, pivot to headful and click:

```ts
// pivot to headed browser (stealth + headed = checkbox clickable)
const headfulBrowser = await chromium.launch({ headless: false });
const headfulContext = await headfulBrowser.newContext({ storageState });
const headfulPage = await headfulContext.newPage();
await headfulPage.goto(url);

// click the turnstile checkbox
await headfulPage.click('iframe[src*="challenges.cloudflare.com"] input[type="checkbox"]');
await headfulPage.waitForTimeout(2000);

// save session and close
await headfulContext.storageState({ path: sessionPath });
await headfulBrowser.close();
```

**expected outcome**: checkbox clicked in headed mode, session saved, resume headless.

**note**: OSS solvers like turnaround still available as fallback but typically unnecessary for managed checkbox mode.

### tier 3: human fallback (puzzle)

if captcha presents interactive puzzle (not just checkbox), human intervention needed:

```ts
console.log('captcha puzzle detected — solve in browser window');
await pivotToHeadfulBrowser({ sessionPath, url });
await waitForUserInput();  // human solves puzzle
```

**expected outcome**: human solves captcha puzzle, session preserved, automation resumes.
