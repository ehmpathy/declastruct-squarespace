# Infrastructure Research: Remote State Caching for Playwright-Based Automation

## Overview

This research documents the infrastructure patterns and techniques for implementing remote state caching in browser automation projects. Since Playwright (like Puppeteer) operations are inherently slow due to browser interactions, implementing proper caching is critical for performance.

The patterns documented here are derived from the production implementation in `svc-ads-online-search`, which uses Puppeteer for Google Ads UI automation.

---

## Core Libraries

### 1. with-remote-state-cache

**Repository:** https://github.com/ehmpathy/with-remote-state-cache
**Version:** (upcoming rename from with-remote-state-caching)

A wrapper library that provides declarative, domain-aware caching for remote-state resources.

**Key Features:**
- Declarative cache invalidation, updates, and optimistic updates
- Customizable cache data store (localstorage, s3, dynamodb, on-disk, etc)
- Wrapper pattern for simple and clean usage
- Built on `with-simple-cache`

### 2. with-simple-cache

**Repository:** https://github.com/ehmpathy/with-simple-cache
**Version:** 0.15.1

A foundational caching wrapper that makes it simple to add caching to any function.

**Key Features:**
- Automatic cache key serialization from function arguments
- Sync and async cache support
- Extendable caching with explicit invalidation

### 3. simple-in-memory-cache

**Repository:** https://github.com/ehmpathy/simple-in-memory-cache
**Version:** 0.4.0

In-memory cache with time-based expiration policies for browser and nodejs.

### 4. Bottleneck

**npm:** bottleneck
**Version:** 2.19.5

Distributed task scheduler and rate limiter for controlling concurrency.

## Architecture Pattern

### Remote State Caching Context

The central pattern establishes a shared context for all queries and mutations to interact within:

```ts
import { serialize, deserialize } from 'domain-objects';
import {
  createRemoteStateCachingContext,
  RemoteStateCache,
} from 'with-remote-state-cache';

export const {
  withRemoteStateQueryCache,
  withRemoteStateMutationRegistration,
} = createRemoteStateCachingContext<
  [any, SquarespacePlaywrightClientAgentContext],
  RemoteStateCache
>({
  // Grab the cache from the agent options
  cache: ({ fromInput }) => fromInput[1].agentOptions.remoteState.cache,

  serialize: {
    // Serialize the cache key safely (no credentials, no collisions, but observable)
    key: ({ forInput }) => {
      const key = [
        // Namespace to the account
        forInput[1].agentOptions.account.id,

        // Preview of key-value pairs for observability
        JSON.stringify(forInput[0])
          .replace(/[{}[\]:,]/gi, '_')
          .replace(/[^0-9a-z_]/gi, '')
          .replace(/__+/g, '_')
          .slice(0, 100)
          .replace(/^_/, '')
          .replace(/_$/, ''),

        // Unique hash suffix
        toHashSha256Sync(JSON.stringify(forInput[0])),

        // Serialization version
        'v1',
      ].join('.');
      return key;
    },

    // Serialize domain objects losslessly
    value: (output) => serialize(output, { lossless: true }),
  },

  deserialize: {
    // Deserialize with domain object type information
    value: (cached) => {
      return deserialize(cached, {
        with: [
          SquarespaceDomain,
          SquarespaceDnsRecord,
          SquarespaceDomainContact,
          // ... other domain objects
        ],
      });
    },
  },
});
```

## Query Caching Pattern

### Basic Query with Caching

Wrap read operations with `withRemoteStateQueryCache` to automatically cache responses:

```ts
const listDomains = async (
  { filter, limit }: { filter?: { status?: string }; limit?: number },
  context: SquarespacePlaywrightClientAgentContext,
): Promise<SquarespaceDomain[]> => {
  // Browser automation logic to fetch domains
  // ...
  return domains;
};

const { execute, addTrigger } = withRemoteStateQueryCache(listDomains, {});

export {
  execute as listDomains,
  addTrigger as addTriggerToListDomains,
};
```

**Behavior:**
- First call: Executes browser automation, caches result
- Subsequent calls with same input: Returns cached result instantly
- Cache key is automatically derived from input arguments

## Mutation Registration Pattern

### Registering Mutations

Mutations must be registered to enable cache invalidation triggers:

```ts
import { withRemoteStateMutationRegistration } from './withRemoteStateCaching';
import { withNewLoggedInBrowserPage } from '../access/withNewLoggedInBrowserPage';

const unlockDomain = withNewLoggedInBrowserPage(
  async (
    { domainName }: { domainName: string },
    context: SquarespacePlaywrightClientPageContext,
  ): Promise<void> => {
    const { page } = context;
    // Browser automation to unlock domain
    // ...
  },
);

const registration = withRemoteStateMutationRegistration(unlockDomain, {
  name: 'unlockDomain',
});
const { execute } = registration;

export {
  execute as unlockDomain, // Export the registered execute to trigger invalidations
};
```

## Cache Invalidation Triggers

### Invalidating Queries on Mutation

Connect mutations to queries they affect using `addTrigger`:

```ts
// When unlockDomain is called, invalidate relevant listDomains cache entries
addTriggerToListDomains({
  invalidatedBy: {
    mutation: registration, // The unlockDomain registration
    affects: ({ mutationInput, cachedQueryKeys }) => {
      return {
        inputs: [
          // Invalidate queries that would have returned this domain
          [
            { filter: { name: mutationInput[0].domainName } },
            mutationInput[1], // Pass through the context
          ],
          // Invalidate unfiltered queries (would include this domain)
          [
            {},
            mutationInput[1],
          ],
        ],
        keys: cachedQueryKeys.filter((key) =>
          key.includes(mutationInput[0].domainName),
        ),
      };
    },
  },
});
```

### Complex Multi-Query Invalidation

A single mutation can invalidate multiple queries:

```ts
// Mutation that creates a DNS record affects multiple queries
addTriggerToListDomains({
  invalidatedBy: {
    mutation: addDnsRecordRegistration,
    affects: ({ mutationInput, cachedQueryKeys }) => ({
      inputs: [
        // Invalidate queries for this specific domain
        [
          { filter: { name: mutationInput[0].domainName } },
          mutationInput[1],
        ],
      ],
      keys: cachedQueryKeys.filter((key) =>
        key.includes(mutationInput[0].domainName),
      ),
    }),
  },
});

// Also invalidate DNS records queries
addTriggerToListDnsRecords({
  invalidatedBy: {
    mutation: addDnsRecordRegistration,
    affects: ({ mutationInput, cachedQueryKeys }) => ({
      inputs: [
        [{}, mutationInput[1]], // Invalidate all DNS record queries
      ],
      keys: [
        ...cachedQueryKeys.filter((key) => key.includes('limit')),
        ...cachedQueryKeys.filter((key) =>
          key.includes(mutationInput[0].domainName),
        ),
      ],
    }),
  },
});
```

## Browser & Page Management

### Agent Options Structure

Define comprehensive options for browser management:

```ts
import Bottleneck from 'bottleneck';
import { Page } from 'playwright';
import { SimpleInMemoryCache } from 'simple-in-memory-cache';
import { RemoteStateCache } from 'with-remote-state-cache';
import { SimpleCache } from 'with-simple-cache';

export interface SquarespacePlaywrightClientAgentOptions {
  // Account to operate as
  account: {
    id: string;
    email: string;
  };

  // Login credentials
  credentials: {
    email: string;
    password: string;
    totp?: { secret: string };
  };

  // Browser management options
  browser: {
    // Cache for reusing browser instances
    cache: SimpleInMemoryCache<{
      browser: Browser;
      close: () => Promise<void>;
    }>;

    // Bottleneck to limit concurrent browser operations
    bottleneck: Bottleneck;

    // Optional existing browser endpoint
    existingBrowserWSEndpoint?: string;
  };

  // Request caching options
  request: {
    cache: SimpleCache<string>;
  };

  // Remote state caching options
  remoteState: {
    cache: RemoteStateCache;
  };
}
```

### withNewLoggedInBrowserPage Wrapper

A wrapper that ensures proper page lifecycle management:

```ts
import { Page } from 'playwright';
import { withExtendableCaching } from 'with-simple-cache';

export const withNewLoggedInBrowserPage = <
  TInput,
  TContext extends {
    page: Page;
    agentOptions: SquarespacePlaywrightClientAgentOptions;
  },
  TOutput,
>(
  logic: (input: TInput, context: TContext) => Promise<TOutput>,
  options?: {
    /**
     * Allow specifying a pagekey for reusing a page across requests.
     *
     * Relevance:
     * - SPAs with heavy initial page loads benefit from page reuse
     * - Subsequent client-side navigation is fast
     * - Operations comfortable starting where another left off can share pages
     */
    reusePageKey?: string;
  },
) => {
  return async (
    input: TInput,
    context: Omit<TContext, 'page'>,
  ): Promise<TOutput> => {
    // Execute within the browser bottleneck (one operation at a time)
    return context.agentOptions.browser.bottleneck.schedule(async () => {
      // Get or create a page
      const getNewPage = options?.reusePageKey
        ? withExtendableCaching(getNewLoggedInBrowserPage, {
            cache: context.agentOptions.browser.cache,
            serialize: {
              key: (input) =>
                [
                  options.reusePageKey,
                  JSON.stringify({ credentials: context.agentOptions.credentials }),
                  context.agentOptions.account.id,
                ].join('.'),
            },
          })
        : {
            execute: () => getNewLoggedInBrowserPage(context.agentOptions),
            invalidate: undefined,
          };

      const page = await getNewPage.execute(context.agentOptions);

      try {
        await page.bringToFront(); // Ensure page is active
        const result = await logic(input, { ...context, page } as TContext);
        if (!options?.reusePageKey) await page.close();
        return result;
      } catch (error) {
        // Invalidate cached page on error
        if (getNewPage.invalidate)
          getNewPage.invalidate({ forInput: [context.agentOptions] });
        await page.close();
        throw error;
      }
    });
  };
};
```

**Key Benefits:**
1. **Concurrency Control:** Bottleneck ensures only one operation runs at a time
   - Prevents resource hogging
   - Avoids suspicious activity (parallel automation)
   - Prevents operations from interfering with each other
2. **Page Lifecycle:** Properly creates and closes pages
3. **Page Reuse:** `reusePageKey` option allows sharing pages between operations
4. **Error Recovery:** Invalidates cached pages on error

## Browser Instance Management

### Find or Create Logged-in Browser

Cache browser instances to avoid repeated login overhead:

```ts
import { withSimpleCaching } from 'with-simple-cache';

const createBrowserAndLogin = async ({
  credentials,
  existingBrowserWSEndpoint,
}: {
  credentials: SquarespaceCredentials;
  existingBrowserWSEndpoint?: string;
}) => {
  // Create or connect to browser
  const { browser, close } = existingBrowserWSEndpoint
    ? await connectToExistingBrowser({ existingBrowserWSEndpoint })
    : await getHeadlessBrowser();

  // Perform login
  await loginBrowserToSquarespace({ browser, credentials });

  return { browser, close };
};

export const findOrCreateLoggedInBrowser = (
  agentOptions: SquarespacePlaywrightClientAgentOptions,
) => {
  // Wrap with caching using the agent options' browser cache
  const createBrowserAndLoginWithCaching = withSimpleCaching(
    createBrowserAndLogin,
    { cache: agentOptions.browser.cache },
  );

  return createBrowserAndLoginWithCaching({
    credentials: agentOptions.credentials,
    existingBrowserWSEndpoint: agentOptions.browser.existingBrowserWSEndpoint,
  });
};
```

## Complete Usage Example

### Setting Up the Agent Context

```ts
import Bottleneck from 'bottleneck';
import { createCache } from 'simple-in-memory-cache';
import { createOnDiskCache } from 'simple-on-disk-cache';

export const createSquarespaceAgentOptions = (
  credentials: SquarespaceCredentials,
  account: SquarespaceAccount,
): SquarespacePlaywrightClientAgentOptions => ({
  account,
  credentials,
  browser: {
    cache: createCache(), // In-memory cache for browser instances
    bottleneck: new Bottleneck({ maxConcurrent: 1 }),
  },
  request: {
    cache: createCache(),
  },
  remoteState: {
    cache: createOnDiskCache({
      directory: { path: '.cache/squarespace-remote-state' },
    }),
  },
});
```

### Implementing a Cached Query

```ts
// listDomains.ts
import { withRemoteStateQueryCache } from '../performance/withRemoteStateCache';
import { withNewLoggedInBrowserPage } from '../access/withNewLoggedInBrowserPage';

const listDomainsImpl = async (
  { filter }: { filter?: { status?: string } },
  context: SquarespacePlaywrightClientAgentContext,
): Promise<SquarespaceDomain[]> => {
  // Implementation using Playwright
  const { page } = context;
  await page.goto('https://account.squarespace.com/domains');
  // ... scrape domain list
  return domains;
};

const listDomainsWithPage = withNewLoggedInBrowserPage(listDomainsImpl);

const { execute, addTrigger } = withRemoteStateQueryCache(
  listDomainsWithPage,
  {},
);

export {
  execute as listDomains,
  addTrigger as addTriggerToListDomains,
};
```

### Implementing a Mutation with Invalidation

```ts
// unlockDomain.ts
import { withRemoteStateMutationRegistration } from '../performance/withRemoteStateCache';
import { withNewLoggedInBrowserPage } from '../access/withNewLoggedInBrowserPage';
import { addTriggerToListDomains } from './listDomains';
import { addTriggerToGetDomain } from './getDomain';

const unlockDomainImpl = async (
  { domainName }: { domainName: string },
  context: SquarespacePlaywrightClientPageContext,
): Promise<void> => {
  const { page } = context;
  await page.goto(`https://account.squarespace.com/domains/${domainName}`);
  // ... toggle lock off
};

const unlockDomainWithPage = withNewLoggedInBrowserPage(unlockDomainImpl);

const registration = withRemoteStateMutationRegistration(unlockDomainWithPage, {
  name: 'unlockDomain',
});

export const { execute: unlockDomain } = registration;

// Invalidate listDomains cache when a domain is unlocked
addTriggerToListDomains({
  invalidatedBy: {
    mutation: registration,
    affects: ({ mutationInput, cachedQueryKeys }) => ({
      inputs: [
        [{}, mutationInput[1]], // Invalidate unfiltered queries
      ],
      keys: cachedQueryKeys.filter((key) =>
        key.includes(mutationInput[0].domainName),
      ),
    }),
  },
});

// Invalidate getDomain cache for the specific domain
addTriggerToGetDomain({
  invalidatedBy: {
    mutation: registration,
    affects: ({ mutationInput }) => ({
      inputs: [
        [{ domainName: mutationInput[0].domainName }, mutationInput[1]],
      ],
      keys: [],
    }),
  },
});
```

## Key Patterns Summary

1. **Query Caching:** Wrap all read operations with `withRemoteStateQueryCache`
2. **Mutation Registration:** Register all write operations with `withRemoteStateMutationRegistration`
3. **Invalidation Triggers:** Connect mutations to affected queries via `addTrigger`
4. **Page Wrapper:** Use `withNewLoggedInBrowserPage` for proper page lifecycle
5. **Bottleneck:** Limit concurrent browser operations to prevent issues
6. **Browser Caching:** Cache logged-in browser instances to avoid repeated logins
7. **Page Reuse:** Use `reusePageKey` for SPA-heavy pages where initial load is expensive

## Playwright vs Puppeteer Considerations

The patterns above were developed for Puppeteer but apply directly to Playwright:

| Puppeteer | Playwright |
|-----------|------------|
| `Browser` from `puppeteer-core` | `Browser` from `playwright` |
| `Page` from `puppeteer-core` | `Page` from `playwright` |
| `browser.newPage()` | `browser.newPage()` |
| `page.goto()` | `page.goto()` |
| `page.waitForSelector()` | `page.waitForSelector()` |
| `page.close()` | `page.close()` |
| `page.bringToFront()` | `page.bringToFront()` |

Playwright offers additional benefits:
- Built-in auto-waiting
- Multiple browser engine support (Chromium, Firefox, WebKit)
- Better debugging tools
- Parallel test execution support
