# playtest: declastruct-squarespace domain transfer

## prerequisites

### credentials

provision squarespace credentials via keyrack:

```sh
chmod +x ./.agent/repo=.this/role=any/skills/keyrack.fill.sh
./.agent/repo=.this/role=any/skills/keyrack.fill.sh --env test
```

or set environment variables directly:

```sh
export SQUARESPACE_EMAIL='your@email.com'
export SQUARESPACE_PASSWORD='your-password'
export SQUARESPACE_TOTP_SECRET='your-totp-secret'
```

### dependencies

```sh
npm install
```

### test account

- squarespace account with at least one domain
- 2fa configured as authenticator app (totp) — sms and passkey 2fa are NOT supported

## manual verification: transfer code request

**WARNING: this will trigger a real transfer code email**

only run this if you intend to transfer a domain.

### steps

1. pick a test domain you want to transfer
2. create a test file:

```ts
// test-transfer.ts
import { getDeclastructSquarespaceProvider } from './src/contract/sdks';

const main = async () => {
  const provider = getDeclastructSquarespaceProvider({
    account: { id: 'test', email: process.env.SQUARESPACE_EMAIL! },
    credentials: {
      email: process.env.SQUARESPACE_EMAIL!,
      password: process.env.SQUARESPACE_PASSWORD!,
      totpSecret: process.env.SQUARESPACE_TOTP_SECRET,
    },
  });

  // unlock domain first
  await provider.daos.DeclaredSquarespaceDomainRegistration.set.upsert({
    entity: { name: 'your-domain.com', isLocked: false, dnssecEnabled: false },
  }, provider.context);

  // request transfer code
  await provider.daos.DeclaredSquarespaceDomainTransferRequest.set.findsert({
    entity: { domain: { name: 'your-domain.com' } },
  }, provider.context);

  console.log('transfer code requested - check your email');
};

main();
```

3. run: `npx tsx test-transfer.ts`
4. check email for transfer code

### pass criteria

- domain unlocked successfully
- transfer code email received

### fail criteria

- unlock fails with 60-day lock error
- transfer code request fails
- no email received

## overall pass/fail

### pass if

- all happy path tests pass
- totp 2fa works automatically
- session reuse works
- domain enumeration returns real data
- stealth plugin active (navigator.webdriver = false)
- captcha detection works
- headful checkbox click works

### fail if

- any happy path test fails
- authentication requires manual intervention
- ui selectors have drifted (selector errors)
- empty data returned when domains exist
- stealth plugin not active
- captcha detection fails
- headful pivot loses session

- integration tests require real squarespace credentials
- tests hit real squarespace ui via playwright
- run tests sequentially to avoid rate limits
