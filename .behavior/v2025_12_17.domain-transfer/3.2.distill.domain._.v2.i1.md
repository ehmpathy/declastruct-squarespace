# Declastruct Squarespace Domain - Distilled Design v2

**Extends:** `3.2.distill.domain._.v1.i1.md`
**New in v2:** Remote state caching infrastructure for performance optimization

---

## 1. Extended Use Cases

### UC6: Efficient Repeated Queries (NEW)
**Goal:** Avoid redundant browser automation when querying same data multiple times.
**Actor:** Automation script performing batch operations
**Flow:**
1. First query executes browser automation, result is cached
2. Subsequent identical queries return cached result instantly
3. Cache invalidates automatically when mutations affect the data
4. Cache persists across process restarts (on-disk)

### UC7: Concurrent Operation Safety (NEW)
**Goal:** Prevent race conditions and resource conflicts during batch processing.
**Actor:** Automation orchestrator running multiple operations
**Flow:**
1. All browser operations go through Bottleneck rate limiter
2. Only one browser operation runs at a time
3. Page reuse where appropriate (same domain detail page)
4. Proper cleanup on errors

### 2.2 Caching Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                     Domain Operations                            │
├─────────────────────────────────────────────────────────────────┤
│  CACHED QUERIES (hit browser, populate cache):                   │
│  getAllDomains ────► withRemoteStateQueryCache ──► Cache Layer  │
│  getAllDnsRecords ─► withRemoteStateQueryCache ──► Cache Layer  │
│                                                                  │
│  DELEGATING QUERIES (filter cached data in-memory):             │
│  getOneDomain ─────► getAllDomains ──► filter ──► result        │
│  getOneDnsRecord ──► getAllDnsRecords ► filter ──► result       │
├─────────────────────────────────────────────────────────────────┤
│  MUTATIONS (invalidate affected getAll caches):                  │
│  setDomain ────────► withRemoteStateMutationRegistration        │
│  setDnsRecord ─────► withRemoteStateMutationRegistration        │
│  setTransferReq ───► withRemoteStateMutationRegistration        │
│                              │                                   │
│                              ▼                                   │
│                    Invalidation Triggers                         │
│                    (invalidate getAllDomains, getAllDnsRecords)  │
├─────────────────────────────────────────────────────────────────┤
│                     Browser Layer                                │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ Bottleneck (maxConcurrent: 1)                           │    │
│  │    └── withNewLoggedInBrowserPage                       │    │
│  │           └── Page Cache (reusePageKey)                 │    │
│  │                  └── Browser Cache (logged-in session)  │    │
│  └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 getOne Delegates to getAll Pattern

**CRITICAL OPTIMIZATION:** `getOne` operations do NOT have their own cache.
Instead, they delegate to `getAll` and filter in-memory.

```
┌──────────────────────────────────────────────────────────────────┐
│  getOneDomain({ by: { name: 'foo.com' } })                       │
│         │                                                         │
│         ▼                                                         │
│  getAllDomains({})  ◄─── Cache HIT? Return cached list           │
│         │                                                         │
│         │ Cache MISS? Execute browser automation, cache result   │
│         ▼                                                         │
│  allDomains.find(d => d.name === 'foo.com')                      │
│         │                                                         │
│         ▼                                                         │
│  Return single domain or null                                    │
└──────────────────────────────────────────────────────────────────┘
```

**Benefits:**
1. **Cache efficiency:** One cache entry serves all 300+ getOne lookups
2. **Cache coherence:** Single source of truth, no stale individual entries
3. **Simpler invalidation:** Only invalidate `getAllDomains`, not 300+ entries
4. **Faster subsequent queries:** After first `getAll`, all `getOne` calls are O(n) in-memory

**Trade-off:**
- First query fetches all data even if only one item needed
- Acceptable because batch operations will need all data anyway

### 2.4 DNS Records: Per-Domain Scope

**IMPORTANT:** Unlike domains, DNS records can only be fetched per-domain.
The Squarespace UI requires navigating to a specific domain's DNS page.

```
┌──────────────────────────────────────────────────────────────────┐
│  Cache Scope Comparison:                                          │
│                                                                   │
│  DOMAINS:                                                         │
│  getAllDomains({}) ──► Single cache entry for ALL 300+ domains   │
│                                                                   │
│  DNS RECORDS:                                                     │
│  getAllDnsRecords({ by: { domain: 'foo.com' } }) ──► 1 cache entry│
│  getAllDnsRecords({ by: { domain: 'bar.com' } }) ──► 1 cache entry│
│  getAllDnsRecords({ by: { domain: 'baz.com' } }) ──► 1 cache entry│
│                        ... 300 separate cache entries             │
└──────────────────────────────────────────────────────────────────┘
```

**Why this matters:**
- `getAllDomains()` - One browser navigation, one cache entry
- `getAllDnsRecords({ domain })` - One navigation PER domain, separate cache per domain
- For batch DNS export: 300 domains = 300 browser navigations (but each cached)

**Optimization for DNS:**
- `getOneDnsRecord` still delegates to `getAllDnsRecords(domain)` and filters
- Within a single domain, avoids duplicate DNS page scraping
- Invalidation is per-domain (mutation to foo.com only invalidates foo.com's DNS cache)

```typescript
import { RemoteStateCache } from 'with-remote-state-cache';
import { SimpleCache } from 'with-simple-cache';

/**
 * @what - Complete agent options for Squarespace Playwright automation
 * @why - Centralizes all caching, rate limiting, and browser management config
 */
export interface ContextSquarespaceAgentOptions {
  /**
   * Account being operated on
   */
  account: {
    /**
     * Unique identifier for cache namespacing
     */
    id: string;

    /**
     * Account email (for display/logging)
     */
    email: string;
  };

  /**
   * Login credentials for authentication
   */
  credentials: {
    email: string;
    password: string;
    totp?: {
      /**
       * Base32-encoded TOTP secret for 2FA
       */
      secret: string;
    };
  };

  /**
   * Browser instance management
   */
  browser: {
    /**
     * Cache for reusing logged-in browser instances
     * @why - Avoids repeated login overhead
     */
    cache: SimpleInMemoryCache<{
      browser: Browser;
      context: BrowserContext;
      close: () => Promise<void>;
    }>;

    /**
     * Rate limiter for browser operations
     * @why - Prevents concurrent operations that could conflict
     */
    bottleneck: Bottleneck;

    /**
     * Optional existing browser WebSocket endpoint
     * @why - Allows connecting to externally managed browser
     */

  };

  /**
   * Page instance management
   */
  page: {
    /**
     * Cache for reusing pages by key
     * @why - SPAs benefit from page reuse (avoid initial load)
     */
    cache: SimpleInMemoryCache<Page>;
  };

  /**
   * Request-level caching (HTTP responses, etc.)
   */
  request: {
    cache: SimpleCache<string>;
  };

  /**
   * Remote state caching for domain operations
   */
  remoteState: {
    /**
     * Persistent cache for query results
     * @why - Survives process restarts, enables fast repeated queries
     */

  };
```

## 4. Remote State Caching Context

### 4.1 Creating the Caching Context

```typescript
import { serialize, deserialize } from 'domain-objects';
import {
  createRemoteStateCacheContext,
  RemoteStateCache,
} from 'with-remote-state-cache';
import { toHashSha256Sync } from 'hash-fns';

// Import all domain objects for deserialization
import { DeclaredSquarespaceDomainRegistration } from '../domain.objects/DeclaredSquarespaceDomainRegistration';
import { DeclaredSquarespaceDomainDnsRecord } from '../domain.objects/DeclaredSquarespaceDomainDnsRecord';
import { DeclaredSquarespaceDomainTransferRequest } from '../domain.objects/DeclaredSquarespaceDomainTransferRequest';

export const {
  withRemoteStateQueryCache,
  withRemoteStateMutationRegistration,
} = createRemoteStateCacheContext<
  [any, ContextSquarespaceAgent],
  RemoteStateCache
>({
  // Extract cache from agent options
  cache: ({ fromInput }) => fromInput[1].agentOptions.remoteState.cache,

  serialize: {
    // Create a unique, observable cache key
    key: ({ forInput }) => {
      const key = [
        // Namespace to the account
        forInput[1].agentOptions.account.id,

        // Preview of key-value pairs for observability
        JSON.stringify(forInput[0])
          .replace(/[{}[\]:,]/gi, '_')
          .replace(/[^0-9a-z_]/gi, '')
          .replace(/__+/g, '_')
          .slice(0, 100)
          .replace(/^_/, '')
          .replace(/_$/, ''),

        // Unique hash suffix
        toHashSha256Sync(JSON.stringify(forInput[0])),

        // Serialization version (bump on schema changes)
        'v1',
      ].join('.');
      return key;
    },

    // Serialize domain objects losslessly
    value: (output) => serialize(output, { lossless: true }),
  },

  deserialize: {
    // Deserialize with domain object type information
    value: (cached) => {
      return deserialize(cached, {
        with: [
          DeclaredSquarespaceDomainRegistration,
          DeclaredSquarespaceDomainDnsRecord,
          DeclaredSquarespaceDomainTransferRequest,
        ],
      });
    },
  },
});
```

## 5. Browser & Page Management

### 5.1 withNewLoggedInBrowserPage Wrapper

```typescript
import type { Page } from 'playwright';
import { withExtendableCache } from 'with-simple-cache';

/**
 * @what - Wrapper that provides a logged-in browser page to operations
 * @why - Centralizes page lifecycle, concurrency control, and error recovery
 */
export const withNewLoggedInBrowserPage = <
  TInput,
  TOutput,
>(
  logic: (input: TInput, context: ContextSquarespaceAgentPage) => Promise<TOutput>,
  options?: {
    /**
     * Key for page reuse across operations
     * @why - SPAs benefit from keeping page open (avoid initial load)
     * @example - 'domainDetail' for domain detail operations
     */
    reusePageKey?: string;
  },
) => {
  return async (
    input: TInput,
    context: ContextSquarespaceAgent,
  ): Promise<TOutput> => {
    const { agentOptions } = context;

    // Execute within bottleneck (one operation at a time)
    return agentOptions.browser.bottleneck.schedule(async () => {
      // Get or create page (with optional caching + invalidation on error)
      const getPage = options?.reusePageKey
        ? withExtendableCache(getNewLoggedInBrowserPage, {
            cache: agentOptions.page.cache,
            serialize: {
              key: () => [
                options.reusePageKey,
                agentOptions.account.id,
              ].join('.'),
            },
          })
        : {
            execute: () => getNewLoggedInBrowserPage(agentOptions),
            invalidate: undefined,
          };

      const page = await getPage.execute(agentOptions);

      try {
        await page.bringToFront();
        const result = await logic(input, { ...context, page });

        // Close page unless reusing
        if (!options?.reusePageKey) {
          await page.close();
        }

        return result;
      } catch (error) {
        // Invalidate cached page on error (page may be in bad state)
        if (getPage.invalidate) {
          getPage.invalidate({ forInput: [agentOptions] });
        }
        await page.close().catch(() => {}); // Best effort close
        throw error;
      }
    });
  };
};
```

### 5.2 Browser Instance Management

```typescript
import { chromium } from 'playwright';
import { withSimpleCache } from 'with-simple-cache';

/**
 * Create a new browser and perform login
 */
const createBrowserAndLogin = async (
  agentOptions: ContextSquarespaceAgentOptions,
): Promise<{
  browser: Browser;
  context: BrowserContext;
  close: () => Promise<void>;
}> => {
  // Connect to existing or launch new browser
  const browser = agentOptions.browser.existingBrowserWSEndpoint
    ? await chromium.connectOverCDP(agentOptions.browser.existingBrowserWSEndpoint)
    : await chromium.launch({ headless: true });

  // Create browser context
  const context = await browser.newContext({
    userAgent: 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7) ...',
    viewport: { width: 1920, height: 1080 },
  });

  // Perform login
  const loginPage = await context.newPage();
  await performSquarespaceLogin(loginPage, agentOptions.credentials);
  await loginPage.close();

  return {
    browser,
    context,
    close: async () => {
      await context.close();
      await browser.close();
    },
  };
};

 * Get or create a logged-in browser instance

  agentOptions: ContextSquarespaceAgentOptions,

    cache: agentOptions.browser.cache,

      key: (opts) => [

        opts.credentials.email,

};

  agentOptions: ContextSquarespaceAgentOptions,

  const { context } = await findOrCreateLoggedInBrowser(agentOptions);
  return context.newPage();
```

### 6.2 getOneDomain - Delegates to getAllDomains

**IMPORTANT PATTERN:** `getOne` operations should delegate to `getAll` and filter in-memory.
This maximizes cache hits because:
1. If `getAllDomains` was already cached, `getOneDomain` returns instantly
2. If `getOneDomain` is called first, it populates the `getAllDomains` cache for future queries
3. A single cache entry serves all `getOne` lookups

```typescript
import { getAllDomains } from './getAllDomains';

/**
 * Get a single domain by filtering the cached list
 * @pattern - Delegates to getAll to maximize cache hits
 */
export const getOneDomain = async (
  input: {
    by: PickOne<{ unique: RefByUnique<typeof DeclaredSquarespaceDomainRegistration> }>;
  },
  context: ContextSquarespaceAgent,
): Promise<HasReadonly<typeof DeclaredSquarespaceDomainRegistration> | null> => {
  const { name } = input.by.unique;

  // Delegate to getAllDomains (uses cache if available)
  const allDomains = await getAllDomains({}, context);

  // Filter in memory
  const domain = allDomains.find((d) => d.name === name);

  return domain ?? null;
};

// No separate cache needed - leverages getAllDomains cache
// No separate addTrigger needed - invalidation handled by getAllDomains
```

**Why not cache getOneDomain separately?**
- 300+ domains × separate cache entries = cache bloat
- Individual domain pages may have stale data vs list view
- Single `getAllDomains` cache is easier to invalidate correctly

## 7. Mutation Operations with Cache Invalidation

### 7.1 setDomain (unlock/DNSSEC) with Invalidation

```typescript
import { withRemoteStateMutationRegistration } from '../../infra/performance/withRemoteStateCache';
import { addTriggerToGetAllDomains } from './getAllDomains';
// NOTE: No trigger for getOneDomain - it delegates to getAllDomains

const _setDomain = async (
  input: {
    upsert: Pick<DeclaredSquarespaceDomainRegistration, 'name' | 'isLocked' | 'dnssecEnabled'>;
  },
  context: ContextSquarespaceAgentPage,
): Promise<HasReadonly<typeof DeclaredSquarespaceDomainRegistration>> => {
  const { page } = context;
  const { upsert } = input;

  // Navigate to domain
  await page.goto(`https://account.squarespace.com/domains/${upsert.name}`);

  // Get current state
  const current = await scrapeDomainDetail(page);

  // Toggle DNSSEC if needed
  if (current.dnssecEnabled !== upsert.dnssecEnabled) {
    await toggleDnssec(page, upsert.dnssecEnabled);
  }

  // Toggle lock if needed
  if (current.isLocked !== upsert.isLocked) {
    await toggleDomainLock(page, upsert.isLocked);
  }

  // Return updated state
  return scrapeDomainDetail(page);
};

const _setDomainWithPage = withNewLoggedInBrowserPage(_setDomain);
const _setDomainRegistration = withRemoteStateMutationRegistration(_setDomainWithPage, {
  name: 'setDomain',
});

export const { execute: setDomain } = _setDomainRegistration;

/**
 * Invalidate getAllDomains when a domain is modified
 */
addTriggerToGetAllDomains({
  invalidatedBy: {
    mutation: _setDomainRegistration,
    affects: ({ mutationInput, cachedQueryKeys }) => ({
      // Invalidate unfiltered queries
      inputs: [[{}, mutationInput[1]]],
      // Invalidate any cached queries mentioning this domain
      keys: cachedQueryKeys.filter((key) =>
        key.includes(mutationInput[0].upsert.name),
      ),
    }),
  },
});

// getOneDomain automatically sees updated data because it delegates to getAllDomains
```

### 7.2 setDnsRecord with Multi-Query Invalidation

```typescript
const _setDnsRecord = async (
  input: PickOne<{
    findsert: DeclaredSquarespaceDomainDnsRecord;
    upsert: DeclaredSquarespaceDomainDnsRecord;
  }>,
  context: ContextSquarespaceAgentPage,
): Promise<HasReadonly<typeof DeclaredSquarespaceDomainDnsRecord>> => {
  const { page } = context;
  const record = input.findsert ?? input.upsert;

  await page.goto(
    `https://account.squarespace.com/domains/${record.domain.name}/dns`,
  );

  // Check if record exists
  const existing = await findDnsRecord(page, record);

  if (existing && input.findsert) {
    return existing; // findsert: return existing
  }

  if (existing && input.upsert) {
    // upsert: delete then create
    await deleteDnsRecord(page, existing);
  }

  // Create the record
  await addDnsRecord(page, record);

  // Return created record
  return scrapeDnsRecord(page, record);
};

const _setDnsRecordWithPage = withNewLoggedInBrowserPage(_setDnsRecord);
const _setDnsRecordRegistration = withRemoteStateMutationRegistration(_setDnsRecordWithPage, {
  name: 'setDnsRecord',
});

export const { execute: setDnsRecord } = _setDnsRecordRegistration;

/**
 * Invalidate DNS record queries when records change
 */
addTriggerToGetAllDnsRecords({
  invalidatedBy: {
    mutation: _setDnsRecordRegistration,
    affects: ({ mutationInput, cachedQueryKeys }) => {
      const record = mutationInput[0].findsert ?? mutationInput[0].upsert;
      return {
        inputs: [
          // Invalidate queries for this domain
          [{ by: { domain: { name: record.domain.name } } }, mutationInput[1]],
        ],
        keys: cachedQueryKeys.filter((key) =>
          key.includes(record.domain.name),
        ),
      };
    },
  },
});
```

```typescript
/**
 * Get a single transfer request by filtering the cached list
 * @pattern - Delegates to getAll to maximize cache hits
 */
export const getOneTransferRequest = async (
  input: {
    by: PickOne<{ unique: RefByUnique<typeof DeclaredSquarespaceDomainTransferRequest> }>;
  },
  context: ContextSquarespaceAgent,

  const allRequests = await getAllTransferRequests({}, context);

  // Filter in memory
  const request = allRequests.find((r) => r.domain.name === domain.name);

  return request ?? null;
};

// No separate cache - leverages getAllTransferRequests cache
```

## 8. Invalidation Trigger Summary

| Mutation | Invalidates Cache | Why |
|----------|-------------------|-----|
| `setDomain` | `getAllDomains` | Domain state changed (lock/DNSSEC) |
| `setDnsRecord` | `getAllDnsRecords(domain)` | DNS record added/updated |
| `delDnsRecord` | `getAllDnsRecords(domain)` | DNS record removed |
| `setTransferRequest` | `getAllDomains`, `getAllTransferRequests` | Domain may show transfer pending, new transfer created |

**Note:** `getOneDomain`, `getOneDnsRecord`, and `getOneTransferRequest` do NOT have their own cache entries.
They delegate to `getAll*` operations, so invalidating `getAll*` automatically
invalidates all dependent `getOne*` queries.

## 10. Usage Example with Caching

```typescript
import { createSquarespaceAgentOptions } from './domain.operations/context/createSquarespaceAgentOptions';
import { getAllDomains } from './domain.operations/domainRegistration/getAllDomains';
import { getAllDnsRecords } from './domain.operations/domainDnsRecord/getAllDnsRecords';
import { setDomain } from './domain.operations/domainRegistration/setDomain';
import { setTransferRequest } from './domain.operations/domainTransferRequest/setTransferRequest';

async function batchTransferDomains() {
  // Create agent options with caching enabled
  const agentOptions = createSquarespaceAgentOptions({
    account: { id: 'acct_123', email: 'admin@example.com' },
    credentials: {
      email: process.env.SQUARESPACE_EMAIL!,
      password: process.env.SQUARESPACE_PASSWORD!,
      totpSecret: process.env.SQUARESPACE_TOTP_SECRET,
    },
    cacheDirectory: '.cache/squarespace',
  });

  const context = { agentOptions };

  // First call: hits browser, caches result
  console.log('Fetching domains (first call - browser)...');
  const domains = await getAllDomains({}, context);
  console.log(`Found ${domains.length} domains`);

  // Second call: instant from cache
  console.log('Fetching domains (second call - cached)...');
  const domainsCached = await getAllDomains({}, context);
  console.log(`Found ${domainsCached.length} domains (from cache)`);

  // Process each domain
  for (const domain of domains) {
    console.log(`Processing ${domain.name}...`);

    // Export DNS (cached after first call per domain)
    const dnsRecords = await getAllDnsRecords(
      { by: { domain: { name: domain.name } } },
      context,
    );
    await saveDnsRecordsToFile(domain.name, dnsRecords);

    // Skip if locked with 60-day restriction
    if (domain.lockReason) {
      console.log(`  Skipping: ${domain.lockReason}`);
      continue;
    }

    // Unlock and disable DNSSEC (invalidates cache automatically)
    if (domain.isLocked || domain.dnssecEnabled) {
      await setDomain({
        upsert: {
          name: domain.name,
          isLocked: false,
          dnssecEnabled: false,
        },
      }, context);
    }

    // Request transfer code (invalidates cache automatically)
    await setTransferRequest({
      findsert: { domain: { name: domain.name } },
    }, context);

    console.log(`  Transfer code requested for ${domain.name}`);

    // Rate limit between domains
    await sleep(2000);
  }

  // Cleanup
  const { close } = await findOrCreateLoggedInBrowser(agentOptions);
  await close();
}
```

## 12. Implementation Notes

### Cache Invalidation Strategy
- **Conservative:** Invalidate more rather than less
- **Query-specific:** Target exact queries when possible
- **Key-based:** Use key filtering for broad invalidation

### Bottleneck Configuration
- `maxConcurrent: 1` - Only one browser operation at a time
- `minTime: 500` - Minimum 500ms between operations (rate limiting)

### Page Reuse Keys
- `domainsList` - For list operations (scraping all domains)
- `domainDetail` - For single domain operations
- `dnsSettings` - For DNS operations

### Error Recovery
- Invalidate cached pages on error
- Close browser on unrecoverable errors
- Retry logic at operation level (not shown)
