### gap that may need action

**usecase.5 lock validation gap:**
- criteria line 97-100 says: "given('a locked domain') when('user requests transfer code') then('returns error: domain must be unlocked first')"
- code does NOT explicitly check lock status before transfer request
- Squarespace UI blocks the request, but error message is generic automation failure
- **recommendation**: add lock check at start of `setTransferRequest.ts` for explicit error:
  ```ts
  // fetch domain to check lock status
  const domain = await getOneDomain({ by: { unique: { name: domainName } } }, context);
  if (domain?.isLocked) {
    throw new BadRequestError('domain must be unlocked before requesting transfer code', { domain: domainName });
  }
  ```

**verdict:** this is a minor gap since Squarespace UI handles it; explicit check would improve error clarity.
