### gap 2: scraper action tests

**scraper action files lack dedicated tests:**
- `toggleDomainLock.ts` - no dedicated test file
- `toggleDnssec.ts` - no dedicated test file
- `requestTransferCode.ts` - no dedicated test file

**analysis:** these actions are browser-dependent and:
1. cannot be unit tested (would require mocks)
2. are exercised via `setDomain` and `setTransferRequest` operations
3. are validated end-to-end in acceptance tests

**severity:** acceptable - isolated unit tests would violate rule.forbid.remote-boundaries

### gap 3: error scenario tests

**checked for error case coverage:**
- `setDomain.ts` throws `BadRequestError` for domain not found
- `setDomain.ts` throws `BadRequestError` for 60-day lock restriction
- these error paths lack explicit test cases

**severity:** minor - happy path coverage is present, edge cases could be stronger
