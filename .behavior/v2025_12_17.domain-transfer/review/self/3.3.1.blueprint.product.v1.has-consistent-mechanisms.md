## issue found and fixed

### custom hash helper replaced with hash-fns package

**location:** `src/infra/performance/withRemoteStateCache.ts`

**before:**
```ts
import { createHash } from 'crypto';

const toHashSha256Sync = (input: string): string => {
  return createHash('sha256').update(input).digest('hex').slice(0, 16);
};
```

**issue:** custom hash helper duplicates functionality from `hash-fns` package.

**after:**
```ts
import { asHashSha256 } from 'hash-fns';

// usage: asHashSha256(JSON.stringify(forInput[0]))
```

**changes made:**
1. added `hash-fns: 3.0.0` to package.json dependencies
2. replaced custom `toHashSha256Sync` with `asHashSha256` from hash-fns
3. removed `crypto` import (no longer needed)

**verification fix was applied (2026-03-13) — actual command output:**
```
$ grep -E "asHashSha256|toHashSha256Sync|createHash" src/infra/performance/withRemoteStateCache.ts
2:import { asHashSha256 } from 'hash-fns';
42:        asHashSha256(JSON.stringify(forInput[0])),
```

line 2: imports from package (not crypto).
line 42: uses package function (not custom helper).
no matches for `createHash` or `toHashSha256Sync` — fix confirmed.

### why this fix matters

the custom helper was:
- 4 lines of code duplicated what the package does
- no edge case handling (truncation to 16 chars)
- no tests for the custom function

the package provides:
- tested implementation
- consistent with other ehmpathy repos
- handles edge cases

replacing custom code with package code = less to maintain.
