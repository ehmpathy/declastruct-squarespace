## violations found

### violation 1: incorrect primary key definition

**location:** `src/domain.objects/DeclaredSquarespaceDomainDnsRecord.ts` line 51

**code:**
```ts
public static primary = ['domain'] as const;
public static unique = ['domain', 'type', 'host'] as const;
```

**issue:** `primary = ['domain']` means all DNS records for a domain have the same primary key. this is nonsensical - primary keys must uniquely identify ONE record.

**rule:** domain-objects requires primary to be a stable unique identifier

**severity:** blocker - incorrect domain object definition

**recommendation:** change to `primary = ['domain', 'type', 'host']` (same as unique)

### violation 2: bag ref instead of RefByUnique

**location:** `src/access/daos/DeclaredSquarespaceDomainDnsRecordDao.ts` line 35

**code:**
```ts
all: async (
  input: { domainName: string },  // bag ref - plain string
  context: ContextSquarespaceAgent,
)
```

**spec says:** vision line 75: `daos.DeclaredSquarespaceDomainDnsRecord.get.all({ domain: { name } })`

**rule:** `rule.require.dobj.references.devtime` requires `RefByUnique<typeof T>`, not plain strings

**severity:** blocker - violates reference type rules

**recommendation:** change to `input: { domain: RefByUnique<typeof DeclaredSquarespaceDomainRegistration> }`

### violation 3: YAGNI - unrequested DNS operations

**location:** `src/domain.operations/domainDnsRecord/setDnsRecord.ts`, `delDnsRecord.ts`

**wish says:** "get operations to see the dns of those domains"

**implementation has:** `setDnsRecord`, `delDnsRecord`, `findsert`, `upsert` operations

**rule:** `rule.prefer.wet-over-dry` says wait for usage before abstraction

**note:** the code even documents this with `.yagni` comments but implements anyway

**severity:** nitpick - extra code not requested, but documented

### violation 4: barrel exports in domain.objects

**location:**
- `src/domain.objects/index.ts`
- `src/domain.objects/literals/index.ts`
- `src/access/daos/index.ts`

**rule:** rule.forbid.barrel-exports

**issue:** all three files use the re-export pattern (`export { X } from './X'`) which is explicitly forbidden. the rule states:
> banned examples:
> - src/domain.objects/index.ts
> allowed only: in index.ts file, export one object

**severity:** blocker - explicitly banned pattern

**recommendation:** either:
1. remove index.ts files and import directly from source files
2. or restructure to export a single object (like a namespace)

### violation 5: bag refs in domain object construction

**rule:** `rule.forbid.dobj.bagrefs` - forbid untyped inline object literals for domain object references

**locations:**
1. `src/domain.operations/domainDnsRecord/castIntoDeclaredSquarespaceDomainDnsRecord.ts` line 23:
   ```ts
   domain: { name: domainName },  // untyped bag ref
   ```

2. `src/domain.operations/domainTransferRequest/setTransferRequest.ts` line 72:
   ```ts
   domain: { name: domainName },  // untyped bag ref
   ```

3. `src/domain.operations/domainTransferRequest/castIntoDeclaredSquarespaceDomainTransferRequest.ts` line 58:
   ```ts
   domain: { name: raw.domainName },  // untyped bag ref
   ```

4. test files also use bag refs:
   - `DeclaredSquarespaceDomainTransferRequest.test.ts` lines 9, 23
   - `DeclaredSquarespaceDomainDnsRecord.test.ts` lines 9, 31, 48

**issue:** all use `{ name: string }` inline instead of `RefByUnique.as<typeof DeclaredSquarespaceDomainRegistration>({ name })`

**severity:** blocker - violates reference safety rules

**recommendation:** use `RefByUnique.as<typeof DeclaredSquarespaceDomainRegistration>({ name: domainName })`

### violation 6: undocumented type casts

**rule:** `rule.forbid.as-cast` - forbid `as` casts without boundary documentation

**locations:**

1. `src/domain.operations/domainDnsRecord/castIntoDeclaredSquarespaceDomainDnsRecord.ts` line 24:
   ```ts
   type: raw.type.toUpperCase() as DeclaredSquarespaceDomainDnsRecordType,
   ```

2. `src/domain.operations/domainDnsRecord/setDnsRecord.ts` lines 81-88:

**issue:** type casts without documented boundary justification or removal path

**severity:** blocker - `as` casts are forbidden per rule

**recommendation:** use type guards or runtime validation instead of `as` casts; if at boundary, add note that explains why types lack and removal path

### violation 7: blueprint/implementation name mismatch

**rule:** blueprint should reflect actual implementation
**issue:** declares `wsEndpointExtant` but implementation uses a different property name
**severity:** minor (cosmetic, functionally equivalent)
