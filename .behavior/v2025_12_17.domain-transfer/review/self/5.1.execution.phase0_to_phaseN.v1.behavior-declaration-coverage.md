## gaps found and fixed

### gap 1: acceptance test resources lacked dnssecEnabled

**issue:** acceptance test resources declared `isLocked: false` but did not include `dnssecEnabled: false` per blackbox criteria usecase.4.

**file:** `src/contract/sdks/.test/assets/resources.acceptance.ts`

**fix:**
```ts
// before
new DeclaredSquarespaceDomainRegistration({
  ...domain,
  isLocked: false, // unlock for transfer
}),

// after
new DeclaredSquarespaceDomainRegistration({
  ...domain,
  isLocked: false, // unlock for transfer
  dnssecEnabled: false, // disable dnssec for transfer (per blackbox criteria usecase.4)
}),
```

**status:** fixed. acceptance test now covers usecase.4.

### gap 2: unsupported 2FA method detection (usecase.7)

**issue:** blackbox criteria usecase.7 requires specific error messages for unsupported 2FA methods:
- `'totp required, sms not supported'` for SMS 2FA
- `'totp required, passkeys not supported'` for passkey 2FA
- `'authentication failed'` for invalid credentials

implementation only had generic error: `'2FA required but no TOTP secret provided'`

**files:**
- `src/access/sdks/squarespace.via.playwright/selectors/authSelectors.ts`
- `src/access/sdks/squarespace.via.playwright/auth/performSquarespaceLogin.ts`

**fix 1:** added selectors for SMS and passkey 2FA detection
```ts
// authSelectors.ts - added
sms2faPage: {
  container: '[data-test="sms-verification"], [data-test="phone-verification"]',
  phoneInput: 'input[type="tel"], input[name="phone"]',
  sendCodeButton: 'button:has-text("Send"), button:has-text("Text me")',
},
passkey2faPage: {
  container: '[data-test="passkey-verification"], [data-test="security-key"]',
  usePasskeyButton: 'button:has-text("Use passkey"), button:has-text("Security key")',
},
```

**fix 2:** added detection and specific BadRequestError for each 2FA method
```ts
// performSquarespaceLogin.ts - added
const sms2faContainer = await page.$(authSelectors.sms2faPage.container);
if (sms2faContainer) {
  throw new BadRequestError('totp required, sms not supported', {
    hint: 'reconfigure squarespace account to use authenticator app instead of sms',
  });
}

const passkey2faContainer = await page.$(authSelectors.passkey2faPage.container);
if (passkey2faContainer) {
  throw new BadRequestError('totp required, passkeys not supported', {
    hint: 'reconfigure squarespace account to use authenticator app instead of passkey',
  });
}
```

**fix 3:** changed login failure to use BadRequestError with specific message
```ts
// before
throw new Error(`Login failed: ${errorText || 'Unknown error'}`);

// after
throw new BadRequestError('authentication failed', {
  reason: errorText || 'unknown error',
  hint: 'check credentials and try again',
});
```

**status:** fixed. authentication now covers all usecase.7 error scenarios.
