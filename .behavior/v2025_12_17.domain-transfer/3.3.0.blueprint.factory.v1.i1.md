# blueprint: factory

## summary

**factory upgrades**: COMPLETE

| upgrade | status | leverage |
|---------|--------|----------|
| session persistence via storageState | DONE | HIGH |
| persistent browser via wsEndpoint | DONE | HIGH |
| stealth plugin (playwright-extra) | DONE | MEDIUM |
| browser respawn (headless ↔ headful) | DONE | HIGH |
| captcha detection | DONE | HIGH |
| captcha answer (via respawn) | DONE | HIGH |

**key design: shared browser handle with respawn**

the browser handle supports `.respawn({ mode: 'HEADLESS' | 'HEADFUL' })`:
- when captcha is detected, respawn the shared browser in headful mode
- all consumers that have the same handle get the new browser
- after captcha is solved, respawn back to headless
- session is preserved via storageState across respawn

this avoids each captcha function spawn its own browser — a single shared handle governs the browser lifecycle.

**usage**:
```sh
BROWSER_WS_ENDPOINT=ws://localhost:9222/... \
SESSION_STORAGE_PATH=.cache/squarespace-session.json \
npm run test:integration
```

### respawn flow

```
1. respawn({ mode: 'HEADFUL' }) called
   ├─ this.storage.set()                          # save current session to disk
   ├─ close current browser + context
   ├─ launch new browser with requested mode + stealth
   ├─ this.storage.get()                          # restore session into new context
   └─ update handle.browser, handle.context, handle.mode in place

2. all consumers see the new browser
   └─ handle is mutable reference; consumers get new browser automatically

3. after captcha is solved (checkbox clicked or human solved)
   ├─ handle.storage.set()                        # capture captcha-solved session
   └─ handle.respawn({ mode: 'HEADLESS' })
       ├─ storage.set() (auto)                    # saves again (idempotent)
       ├─ close headful browser
       ├─ launch headless browser + stealth
       ├─ storage.get() (auto)                    # restore captcha-solved session
       └─ headless browser has the solved session
```

**note**: respawn() always does storage.set() before close and storage.get() after spawn. the explicit storage.set() after captcha solve ensures the solved session is captured before respawn overwrites it.

### captcha answer flow (via respawn)

```

   └─ detectCaptchaChallenge returns { detected: true, type: 'turnstile' }

2. tier 2: headful checkbox click (if enabled)
   ├─ handle.respawn({ mode: 'HEADFUL' })         # auto: storage.set(), switch, storage.get()
   ├─ clickTurnstileCheckbox (page is now headful)
   ├─ wait for validation (timeout 5s)
   ├─ if solved:
   │   ├─ handle.storage.set()                    # capture captcha-solved session
   │   ├─ handle.respawn({ mode: 'HEADLESS' })    # headless gets solved session
   │   └─ return success
   └─ if failed (puzzle required) → proceed to tier 3

3. tier 3: human fallback (if enabled, puzzle detected)
   ├─ (already in headful mode from tier 2)
   ├─ prompt human: "captcha — solve in browser, press ENTER"

   ├─ handle.storage.set()                        # capture human-solved session
   ├─ handle.respawn({ mode: 'HEADLESS' })
   └─ return { solved: true }

4. retry original operation
   └─ operation completes with solved captcha session (cookies preserved)
```

```ts
import * as fs from 'fs';

const createBrowserAndLogin = async (
  agentOptions: ContextSquarespaceAgentOptions,
): Promise<BrowserAuthSession> => {
  // connect to browser or launch new
  const browser = agentOptions.browser.extantBrowserWSEndpoint
    ? await chromium.connectOverCDP(agentOptions.browser.extantBrowserWSEndpoint)
    : await chromium.launch({ headless: true });

  // restore storageState if file extant
  const storageStatePath = agentOptions.session?.storageStatePath;
  const storageStateExtant = storageStatePath && fs.existsSync(storageStatePath);
  const storageState = storageStateExtant
    ? JSON.parse(fs.readFileSync(storageStatePath, 'utf-8'))
    : undefined;

  // create browser context (with or without storageState)
  const context = await browser.newContext({
    userAgent: '...',
    viewport: { width: 1920, height: 1080 },
    storageState,
  });

  // check session health
  const testPage = await context.newPage();
  const sessionStatus = await checkSessionHealth(testPage);

  if (!sessionStatus.valid) {
    // session invalid — perform login
    await performSquarespaceLogin(testPage, agentOptions.credentials);
  }

  await testPage.close();

  return { browser, context, close: async () => { ... } };
};
```

```ts
export const performSquarespaceLogin = async (
  page: Page,
  credentials: { email: string; password: string; totp?: { secret: string } },
  options?: { storageStatePath?: string },
): Promise<void> => {
  // ... extant login flow ...

  // save storageState after successful login
  if (options?.storageStatePath) {
    const storageState = await page.context().storageState();
    const dir = path.dirname(options.storageStatePath);
    if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
    fs.writeFileSync(options.storageStatePath, JSON.stringify(storageState, null, 2));
```

### 4. detectCaptchaChallenge (NEW)

```ts
// src/access/sdks/squarespace.via.playwright/captcha/detectCaptchaChallenge.ts
import { Page } from 'playwright';

export const detectCaptchaChallenge = async (
  input: { page: Page },
): Promise<{ detected: boolean; type: 'turnstile' | 'challenge' | null }> => {
  const { page } = input;

  // check for cloudflare turnstile iframe
  const turnstileFrame = page.frameLocator('iframe[src*="challenges.cloudflare.com"]');
  const hasTurnstile = await turnstileFrame.locator('body').count().catch(() => 0) > 0;
  if (hasTurnstile) return { detected: true, type: 'turnstile' };

  // check for challenge page (full-page block)
  const challengeDiv = page.locator('div[class*="challenge"], #challenge-running');
  const hasChallenge = await challengeDiv.count() > 0;
  if (hasChallenge) return { detected: true, type: 'challenge' };

  return { detected: false, type: null };
};
```

### 5. answerCaptchaChallenge (NEW)

```ts
// src/access/sdks/squarespace.via.playwright/captcha/answerCaptchaChallenge.ts
import type { Page } from 'playwright';

import type { BrowserAuthSession } from '../browser/genBrowserAuthSession';
import type { CaptchaConfig } from '../../../domain.objects/ContextSquarespaceAgentOptions';

import { detectCaptchaChallenge } from './detectCaptchaChallenge';
import { clickTurnstileCheckbox } from './clickTurnstileCheckbox';
import { waitForHumanSignal } from './waitForHumanSignal';

/**
 * .what - handle captcha challenge via shared browser respawn
 * .why - tiered approach: checkbox click (auto) → human fallback
 */
export const answerCaptchaChallenge = async (
  input: {
    page: Page;
    handle: BrowserAuthSession;
    captchaConfig: CaptchaConfig;
  },
  context: { log: LogMethods },
): Promise<{ solved: boolean; method: 'none' | 'checkbox' | 'human' }> => {
  const { page, handle, captchaConfig } = input;

  // detect captcha
  const detection = await detectCaptchaChallenge({ page });
  if (!detection.detected) return { solved: true, method: 'none' };

  context.log.warn('captcha detected', { type: detection.type });

  // tier 2: headful checkbox click (if enabled)
  if (captchaConfig.checkboxClickEnabled !== false) {
    // respawn shared browser in headful mode
    await handle.respawn({ mode: 'HEADFUL' });

    // get fresh page from new context
    const headfulPage = await handle.context.newPage();
    await headfulPage.goto(page.url());

    // click the checkbox
    const clickResult = await clickTurnstileCheckbox({
      page: headfulPage,
      timeoutMs: captchaConfig.checkboxTimeoutMs ?? 5000,
    });

    if (clickResult.solved) {
      context.log.info('captcha solved via checkbox click');
      // capture solved session before respawn
      await handle.storage.set();
      // respawn back to headless
      await handle.respawn({ mode: 'HEADLESS' });
      return { solved: true, method: 'checkbox' };
    }

    // checkbox failed (puzzle required) → proceed to tier 3
    context.log.warn('checkbox click failed, puzzle detected');
  }

  // tier 3: human fallback (if enabled)
  if (captchaConfig.humanFallbackEnabled !== false) {
    // already in headful mode from tier 2, or respawn now
    if (handle.mode !== 'HEADFUL') {
      await handle.respawn({ mode: 'HEADFUL' });
    }

    // get fresh page from headful context
    const headfulPage = await handle.context.newPage();
    await headfulPage.goto(page.url());

    // prompt human to solve
    await waitForHumanSignal({
      signalMode: captchaConfig.humanSignalMode ?? 'stdin',
      signalFilePath: captchaConfig.humanSignalFilePath,
    });

    context.log.info('captcha solved by human');
    // capture solved session before respawn
    await handle.storage.set();
    // respawn back to headless
    await handle.respawn({ mode: 'HEADLESS' });
    return { solved: true, method: 'human' };
  }

  return { solved: false, method: 'none' };
};
```

### 6. waitForHumanSignal (NEW)

```ts
// src/access/sdks/squarespace.via.playwright/captcha/waitForHumanSignal.ts
import * as fs from 'fs';
import * as readline from 'readline';

/**
 * .what - wait for human to signal captcha solved
 * .why - tier 3 fallback when checkbox click fails (puzzle challenge)
 */
export const waitForHumanSignal = async (
  input: {
    signalMode: 'stdin' | 'file';
    signalFilePath?: string;
  },
): Promise<void> => {
  const { signalMode, signalFilePath } = input;

  console.log('\ncaptcha detected — solve in browser window');

  if (signalMode === 'stdin') {
    console.log('press ENTER after solve...\n');
    await waitForEnter();
  } else if (signalMode === 'file' && signalFilePath) {
    console.log(`create file at ${signalFilePath} when done...\n`);
    await waitForFile(signalFilePath);
  }
};

const waitForEnter = (): Promise<void> => {
  const rl = readline.createInterface({ input: process.stdin, output: process.stdout });
  return new Promise((fulfill) => {
    rl.question('', () => {
      rl.close();
      fulfill();
    });
  });
};

const waitForFile = (filePath: string): Promise<void> => {
  return new Promise((fulfill) => {
    const check = () => {
      if (fs.existsSync(filePath)) {
        fs.unlinkSync(filePath); // clean up signal file
        fulfill();
      } else {
        setTimeout(check, 500);
      }
    };
    check();
  });
};
```

### 7. clickTurnstileCheckbox (NEW)

```ts
// src/access/sdks/squarespace.via.playwright/captcha/clickTurnstileCheckbox.ts
import type { Page } from 'playwright';

import { CAPTCHA_SELECTORS } from '../selectors/captchaSelectors';

/**
 * .what - click turnstile checkbox on page (already headful)
 * .why - checkbox click works in headed mode; page comes from handle after respawn
 * .note - does NOT spawn browser; caller uses handle.respawn() first
 */
export const clickTurnstileCheckbox = async (
  input: { page: Page; timeoutMs?: number },
): Promise<{ solved: boolean }> => {
  const { page, timeoutMs = 5000 } = input;

  try {
    // find and click the turnstile checkbox
    const checkboxFrame = page.frameLocator(CAPTCHA_SELECTORS.turnstileIframe);
    await checkboxFrame.locator(CAPTCHA_SELECTORS.turnstileCheckbox).click();

    // wait for validation
    await page.waitForTimeout(timeoutMs);

    // check if solved (challenge iframe disappears)
    const challengeGone = await page.locator(CAPTCHA_SELECTORS.turnstileIframe).count() === 0;

    return { solved: challengeGone };
  } catch {
    return { solved: false };
  }
};
```

## rationale

per `.behavior/v2025_12_17.domain-transfer/3.1.2.research.external.factory.captcha._.v1.i1.md`:

| upgrade | leverage | status |
|---------|----------|--------|
| session persistence | HIGH | DONE — reduces login frequency |
| stealth plugin | MEDIUM | DONE — reduces captcha trigger rate |
| browser respawn | HIGH | TODO — enables mode switch (headless ↔ headful) |
| captcha detection | HIGH | TODO — detect turnstile and challenge pages |
| headful checkbox click | HIGH | TODO — checkbox clickable in headed mode (no OSS solver) |
| human fallback | HIGH | TODO — prompt human for puzzle challenges |

**tiered approach rationale:**

1. **tier 1: prevention** — stealth plugin reduces captcha trigger rate
2. **tier 2: headful checkbox click** — pivot to headed, click checkbox (works without solver)
3. **tier 3: human fallback** — headful browser for puzzle challenges (rare)

this tiered approach minimizes human intervention while providing reliable recovery when automation is blocked.

factory upgrades unblock safer batch operations.
