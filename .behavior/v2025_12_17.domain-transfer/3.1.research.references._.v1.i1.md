# research: references

## icann (internet corporation for assigned names and numbers)

### transfer policy

**citation [1]**: icann transfer policy
> "Transfer: The process of move a domain name registration between registrars."
> source: https://www.icann.org/resources/pages/transfer-policy-2016-06-01-en

**citation [2]**: icann registrant faqs
> "You'll need the AuthInfo code (also called an Authorization Code, AuthInfo code, Auth-Info Code, or transfer code) to make the transfer, which is a unique code created by a registrar to identify the domain name holder and help prevent unauthorized transfers."
> source: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

**citation [3]**: icann 60-day lock
> "Your registrar may deny a transfer request if the domain name is within 60 days of initial registration... A domain name is within 60 days after being transferred... You cannot transfer a domain name to a different registrar within 60 days of changes to the registrant name, organization or email address."
> source: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

**key concepts established**:
- authorization code (authinfo code)
- 60-day registration lock
- 60-day transfer lock
- 60-day contact change lock
- registrar / registrant distinction

## dns standards (ietf rfcs)

### rfc 1035 - dns specification

**citation [5]**: rfc 1035
> "A resource record (RR) is the basic unit of DNS data."
> source: https://tools.ietf.org/html/rfc1035

**key concepts established**:
- resource record (RR)
- record types: A, CNAME, MX, NS, TXT
- TTL (time to live)

### rfc 3596 - ipv6 dns

**citation [6]**: rfc 3596
> "AAAA records map domain names to IPv6 addresses."
> source: https://tools.ietf.org/html/rfc3596

**key concepts established**:
- AAAA record type

### rfc 4033-4035 - dnssec

**citation [7]**: rfc 4033
> "DNSSEC provides origin authentication and integrity for DNS data."
> source: https://tools.ietf.org/html/rfc4033

**key concepts established**:
- DS record (delegation signer)
- DNSKEY record
- dnssec enable/disable before transfer
