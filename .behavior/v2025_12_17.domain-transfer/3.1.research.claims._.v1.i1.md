# research: claims

## squarespace domain management

### [FACT] squarespace has no public domain management api

**citation [1]**: squarespace developer documentation
> "Squarespace's available API permissions include Orders, Forms, Inventory, and Transactions."
> source: https://developers.squarespace.com/commerce-apis/overview

domain management is not listed among available api permissions. all domain operations require web ui interaction.

### [FACT] squarespace acquired google domains

**citation [2]**: google cloud documentation
> "After Squarespace acquired Google Domains, Google Cloud became a reseller of Squarespace Domains... when you buy new domains using Cloud Domains after migration, the registrar of record is Squarespace."
> source: https://docs.cloud.google.com/domains/docs/faq

this explains why many domains in squarespace may have originated from google domains migration.

## domain transfer requirements

### [FACT] 60-day lock applies to newly registered or transferred domains

**citation [3]**: icann transfer policy
> "Your registrar may deny a transfer request if the domain name is within 60 days of initial registration... A domain name is within 60 days after being transferred."
> source: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

### [FACT] 60-day lock applies after registrant contact changes

**citation [4]**: icann transfer policy
> "You cannot transfer a domain name to a different registrar within 60 days of making changes to the registrant name, organization or email address."
> source: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

### [FACT] authorization code is required for domain transfer

**citation [5]**: icann faq
> "You'll need the AuthInfo code (also called an Authorization Code, AuthInfo code, Auth-Info Code, or transfer code) to make the transfer, which is a unique code created by a registrar to identify the domain name holder and help prevent unauthorized transfers."
> source: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

### [FACT] domain must be unlocked before transfer

**citation [6]**: squarespace help center
> "If your domain is locked, which blocks the transfer, log into your domain provider and unlock the domain."
> source: https://support.squarespace.com/hc/en-us/articles/206542037-Transferring-a-domain-to-a-Squarespace-site

### [FACT] transfers take 1-15 days

**citation [7]**: squarespace help center
> "Transferring a domain to Squarespace can take anywhere from one to 15 days."
> source: https://support.squarespace.com/hc/en-us/articles/115011719247-Domain-transfer-timing

### [FACT] authorization code may be emailed or displayed

**citation [8]**: squarespace forum
> "You will either see the code on the screen or it will be emailed to the administrative contact of the domain."
> source: https://support.squarespace.com/hc/en-us/articles/205812338-Transferring-a-domain-away-from-Squarespace

## dnssec considerations

### [FACT] dnssec should be disabled before transfer

**citation [9]**: cloudflare community
> "The reason why it's recommended to remove DNSSEC before transfer is to allow caches to timeout with the old DS record to expire."
> source: https://community.cloudflare.com/t/dnssec-questions-about-transferring-very-important-domain-from-google-domains-to-cloudflare/453399

### [FACT] dnssec propagation requires 48 hours

**citation [10]**: hacker news
> "You would usually need to disable DNSSEC, wait 24h, transfer, and then wait for..."
> source: https://news.ycombinator.com/item?id=31409361

### [FACT] leaving dnssec enabled during transfer causes connectivity issues

**citation [11]**: cloudflare dns docs
> "If you are onboarding an extant domain to Cloudflare, make sure DNSSEC is disabled at your registrar... Otherwise, your domain will experience connectivity errors when you change your nameservers."
> source: https://developers.cloudflare.com/dns/dnssec/

## squarespace-specific claims

### [FACT] squarespace domain operations require web ui

**citation [16]**: squarespace help center
> "To obtain the authorization code from Squarespace, log in to your Squarespace account and go to Settings > Domains, select the domain you wish to transfer, unlock the domain (if not already done), scroll down to the Authorization Code section."
> source: https://support.squarespace.com/hc/en-us/articles/205812338-Transferring-a-domain-away-from-Squarespace

### [KHUE] does squarespace rate limit domain management operations?

no official documentation found on rate limits for domain ui operations. empirical testing suggests sequential operations with delays avoid triggering defensive measures.

### [KHUE] what registrar types are present for squarespace domains?

domains in squarespace may have different registrar types based on origin:
- squarespace_domains_llc (native squarespace registrations)
- tucows (google domains migration)
- key_systems (other migrations)

### [SUMP] all squarespace registrar types have same transfer ui flow

assumption based on implementation. no documentation confirming or denying differences between registrar types.

### [SUMP] squarespace session timeout is several hours

assumption: long batches (300 domains) may require session refresh. no official documentation on session lifetime.
