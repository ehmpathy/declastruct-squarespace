# Implementation Blueprint: Declastruct Squarespace Domain Transfer

## Overview

This blueprint describes how to implement the Squarespace domain transfer automation system as defined in:
- **Wish:** `.behavior/v2025_12_17.domain-transfer/0.wish.md`
- **Research:** `.behavior/v2025_12_17.domain-transfer/3.1.research.domain._.v1.i1.md`
- **Infrastructure:** `.behavior/v2025_12_17.domain-transfer/3.1.research.infra.remote_state_cache.v1.i1.md`
- **Distillation:** `.behavior/v2025_12_17.domain-transfer/3.2.distill.domain._.v2.i1.md` (preferred)

**Goal:** Automate the transfer of 300+ domains from Squarespace to Cloudflare via Playwright-based UI automation.

```typescript
// scrapeDomainsList.ts
export const scrapeDomainsList = async (
  page: Page,
): Promise<HasReadonly<typeof DeclaredSquarespaceDomainRegistration>[]> => {
  // Wait for domain rows to load
  await page.waitForSelector(SELECTORS.domainRow);

  // Handle pagination/infinite scroll for 300+ domains
  await scrollToLoadAllDomains(page);

  // Extract data from each row
  const rows = await page.$$(SELECTORS.domainRow);
  const domains = await Promise.all(rows.map(async (row) => {
    const name = await row.$eval(SELECTORS.domainName, el => el.textContent);
    const status = await row.$eval(SELECTORS.domainStatus, el => el.textContent);
    // ... extract other fields
    return castIntoDeclaredSquarespaceDomainRegistration({ name, status, ... });
  }));

  return domains;
};
```

**Action implementation pattern:**

```typescript
// toggleDomainLock.ts
export const toggleDomainLock = async (
  page: Page,
  targetLockState: boolean,
): Promise<void> => {
  // Find the lock toggle
  const toggle = await page.$(SELECTORS.domainLockToggle);
  const currentState = await toggle?.getAttribute('aria-checked') === 'true';

  if (currentState === targetLockState) return; // Already in desired state

  // Click toggle
  await toggle?.click();

  // Handle reauthentication if prompted
  await handleReauthenticationIfPresent(page);

  // Wait for confirmation
  await page.waitForSelector(SELECTORS.lockChangeConfirmation);
};
```

   const _getAllDomainsWithPage = withNewLoggedInBrowserPage(_getAllDomains, { reusePageKey: 'domainsList' });

## Key Architectural Decisions

### 1. getOne Delegates to getAll

**Decision:** `getOne` operations delegate to `getAll` and filter in-memory.

**Rationale:**
- Single cache entry serves 300+ lookups
- Simpler invalidation (only invalidate `getAll`)
- Cache coherence (no stale individual entries)

### 2. Per-Domain DNS Caching

**Decision:** DNS records are cached per-domain, not globally.

**Rationale:**
- Squarespace UI requires navigating to each domain's DNS page
- Cannot fetch all DNS records in one operation
- Invalidation is scoped to affected domain only

### 3. Browser Singleton with Bottleneck

**Decision:** One browser operation at a time via Bottleneck.

**Rationale:**
- Prevents resource conflicts
- Avoids suspicious parallel activity
- Ensures operations don't interfere with each other

### 4. On-Disk Cache for Remote State

**Decision:** Use `simple-on-disk-cache` for query results.

**Rationale:**
- Survives process restarts
- Critical for batch operations spanning hours
- Reduces browser automation overhead

## Risk Mitigation

### UI Changes

**Risk:** Squarespace may change their UI, breaking selectors.

**Mitigation:**
- Centralize all selectors in dedicated files
- Use stable selectors (data-testid, aria-labels)
- Implement selector fallback chains
- Regular acceptance test runs to detect breakage

### Rate Limiting

**Risk:** Squarespace may rate-limit or block automation.

**Mitigation:**
- Bottleneck with `minTime: 500ms` between operations
- Configurable delays between domain operations
- Realistic user-agent and viewport
- Consider randomized delays

### Session Expiry

**Risk:** Browser session may expire mid-batch.

**Mitigation:**
- Cache invalidation on auth errors
- Re-login automatically on session expiry
- Monitor for login redirects during operations

### 60-Day Locks

**Risk:** Domains may be locked and untransferable.

**Mitigation:**
- Report locked domains with reason
- Skip locked domains in batch operations
- Track and retry when lock expires

## Success Criteria

1. **Enumerate domains:** List all 300+ domains with full metadata
2. **Export DNS:** Save DNS records for each domain to files
3. **Prepare for transfer:** Unlock domains and disable DNSSEC
4. **Request transfer codes:** Trigger auth code emails for eligible domains
5. **Track progress:** Report status of each domain in batch operation
6. **Efficient caching:** Repeated queries return instantly from cache
7. **Resilient execution:** Handle errors gracefully, continue batch
