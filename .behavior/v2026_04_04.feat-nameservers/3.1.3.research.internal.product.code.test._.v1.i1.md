# research: internal test code patterns — nameservers

## summary

the codebase uses BDD-style tests via `test-fns` with `given`/`when`/`then` semantics. domain objects have unit tests for instantiation and static properties. integration tests use `useBeforeAll` for shared results and `getSampleSquarespaceContext` for credentials.

---

## 1. domain object unit test pattern [REUSE]

### pattern

domain objects are tested for instantiation and static key declarations.

> [1] `src/domain.objects/DeclaredSquarespaceDomainRegistration.test.ts:5-28`
> ```typescript
> describe('DeclaredSquarespaceDomainRegistration', () => {
>   given('a domain registration declaration', () => {
>     when('instantiate via .as() factory', () => {
>       const domain = DeclaredSquarespaceDomainRegistration.as({
>         name: 'example.com',
>         status: 'ACTIVE',
>         // ...
>       });
>
>       then('creates a valid domain object', () => {
>         expect(domain.name).toBe('example.com');
>         expect(domain.status).toBe('ACTIVE');
>       });
>     });
>   });
> });
> ```

> [2] `src/domain.objects/DeclaredSquarespaceDomainRegistration.test.ts:49-72`
> ```typescript
> when('inspect static properties', () => {
>   then('unique key is domain name', () => {
>     expect(DeclaredSquarespaceDomainRegistration.unique).toEqual(['name']);
>   });
>
>   then('readonly properties include status and registrar', () => {
>     expect(DeclaredSquarespaceDomainRegistration.readonly).toContain('status');
>   });
>
>   then('primary key is domain name', () => {
>     expect(DeclaredSquarespaceDomainRegistration.primary).toEqual(['name']);
>   });
> });
> ```

### application

`DeclaredSquarespaceDomainNameservers.test.ts` will follow same pattern:
- test instantiation via `.as()` factory
- test static properties (unique, primary)
- test `nameservers: null` vs `nameservers: [...]` cases

## 3. useBeforeAll pattern [REUSE]

### pattern

shared test results are captured via `useBeforeAll` for reuse across `then` blocks.

> [5] `src/domain.operations/domainRegistration/setDomain.integration.test.ts:31-56`
> ```typescript
> when('setDomain upsert unlocks domain', () => {
>   const result = useBeforeAll(async () => {
>     const domain = await setDomain(
>       { upsert: { name: TEST_DOMAIN, isLocked: false } },
>       context,
>     );
>     return { domain };
>   });
>
>   then('returns the domain', () => {
>     expect(result.domain).toBeDefined();
>   });
>
>   then('domain is a DeclaredSquarespaceDomainRegistration', () => {
>     expect(result.domain).toBeInstanceOf(DeclaredSquarespaceDomainRegistration);
>   });
> });
> ```

### application

nameserver integration tests will use same pattern:
- `useBeforeAll` to call operation once
- multiple `then` blocks for assertions
- return object with named properties

## 5. idempotency test pattern [REUSE]

### pattern

idempotent operations test that repeat calls produce same result.

> [7] `src/access/sdks/squarespace.via.playwright/domainDetail/toggleDomainLock.integration.test.ts:71-101`
> ```typescript
> when('unlock is requested on already unlocked domain (idempotent)', () => {
>   const result = useBeforeAll(async () => {
>     // first unlock
>     await toggleDomainLock({ /* ... */ targetState: 'unlocked' });
>     // second unlock (should be no-op)
>     const secondResult = await toggleDomainLock({ /* ... */ targetState: 'unlocked' });
>     return { secondResult };
>   });
>
>   then('returns success', () => {
>     expect(result.secondResult.success).toBe(true);
>   });
>
>   then('second call was a no-op (no toggle occurred)', () => {
>     expect(result.secondResult.wasNoOp).toBe(true);
>   });
> });
> ```

### application

nameserver tests will verify idempotency:
- set custom nameservers twice, second is no-op
- set null twice, second is no-op
- track `wasNoOp` property in result

## 6. findsert vs upsert test pattern [REUSE]

### pattern

tests verify findsert returns extant without change, upsert always writes.

> [8] `src/domain.operations/domainRegistration/setDomain.integration.test.ts:76-98`
> ```typescript
> when('setDomain findsert with same state (no change needed)', () => {
>   const result = useBeforeAll(async () => {
>     const startTime = Date.now();
>     const domain = await setDomain(
>       { findsert: { name: TEST_DOMAIN, isLocked: true } },
>       context,
>     );
>     const duration = Date.now() - startTime;
>     return { domain, duration };
>   });
>
>   then('returns extant domain', () => {
>     expect(result.domain).toBeDefined();
>     expect(result.domain.isLocked).toBe(true);
>   });
>
>   then('findsert is faster than toggle (no UI interaction)', () => {
>     expect(result.duration).toBeLessThan(30000);
>   });
> });
> ```

### application

nameserver tests will verify:
- findsert returns extant if nameservers match
- upsert always applies changes
- duration comparison to detect no-op (faster when no UI interaction)

### pattern

validation logic is tested in unit tests with clear error expectations.

> [10] implied from setDomain.test.ts pattern
> ```typescript

>   when('nameservers has fewer than 2 entries')
>     then('throws validation error')
>
>   when('nameserver has invalid FQDN format')
>     then('throws validation error')
>
>   when('empty array is provided')
>     then('treats as null (squarespace default)')
> ```

### application

nameserver validation unit tests will cover:
- fewer than 2 nameservers → error
- invalid FQDN format → error
- empty array → treated as null
