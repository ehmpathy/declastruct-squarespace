# blueprint: domain nameservers

## summary

add declarative control over domain nameservers:
- new domain object: `DeclaredSquarespaceDomainNameservers`
- new DAO: `DeclaredSquarespaceDomainNameserversDao` via `genDeclastructDao`
- new operations: `getNameservers`, `setNameservers`
- new scraper: `setNameserversScraper` (GET reuses extant `scrapeDomainDetail`)
- updated selectors: `domainDetailSelectors.ts`

## implementation notes

### validation rules

per RFC 1035 and squarespace constraints:
- minimum 2 nameservers required
- maximum 13 nameservers (RFC 1035 limit for NS records)
- each nameserver must be valid FQDN format
- empty array `[]` treated as `null` (squarespace default)

### nameserver FQDN validation

```typescript
const FQDN_PATTERN = /^(?!-)[a-z0-9-]{1,63}(?<!-)(\.[a-z0-9-]{1,63})*\.?$/i;
```

### null semantics

- `nameservers: null` = squarespace manages nameservers (default)
- `nameservers: [...]` = custom nameservers (user-specified)

this will be documented in JSDoc on the domain object property.
