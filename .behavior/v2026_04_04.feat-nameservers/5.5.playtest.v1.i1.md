# playtest: domain nameservers

## prerequisites

- squarespace test account with credentials configured:
  - `SQUARESPACE_EMAIL`
  - `SQUARESPACE_PASSWORD`
  - `SQUARESPACE_TOTP_SECRET` (optional)
- test domain: `sunshineoceansurferturtles.com` registered in test account
- keyrack unlocked: `rhx keyrack unlock --owner ehmpath --env test`
- persistent browser available (optional, for visual observation): `rhx browser.start --mode HEADFUL --refresh`
  - note: `--refresh` kills any extant browser and starts fresh; without it, tests may reuse an old browser

## sandbox

all file operations target `.temp/playtest/` within the test directory structure (managed by test framework).

- CLI stdout/stderr output is clean and informative

- any test fails unexpectedly
- snapshots mismatch without intentional change

- nameserver changes do not persist (get after set returns different value)
- CLI exits with error on valid input
- idempotency broken (apply â†’ re-plan shows UPDATE instead of KEEP)
